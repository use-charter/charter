# Phase 1 Slice 16 Implementation Plan — Report Delivery (Self-Contained HTML)

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:subagent-driven-development`. Checkbox steps; commit per task. **Depends on Slice 15** (reuses the finding/category model + `charter explain` + design tokens).

**Derived from:** `docs/internal/superpowers/specs/2026-06-02-phase-1-slice-16-design.md` + `docs/internal/decisions/0025-report-delivery-self-contained-html.md`. Spec/ADR own what/why; this owns how. Disagreement → spec/ADR win.

**Design source of truth:** `docs/internal/designs/charter-html-report.html` (inspiration, not copy) + `docs/internal/designs/DESIGN-TOKENS.md` (colors + fonts). Build the visual layer with the installed `.agents/skills/` design set (full map in `DESIGN-TOKENS.md`): **`frontend-design`** (lead), **`web-design-guidelines`** (WCAG 2.2 AA gate), **`design-system-patterns`** + **`visual-design-foundations`** (tokens/type/color), **`interaction-design`** (microinteractions/feedback), **`design-taste-frontend`**, **`high-end-visual-design`**, **`ui-ux-pro-max`** (respect the token system; **screenshot-verify**); **`web-component-design`** for vanilla-HTML structure/composition guidance only.

**Branch & policy:** work on `main`; commit per task; push the whole slice once `moon run :check` is green. GPG-signed.

**Goal:** `charter report --format html` → one self-contained, offline, WCAG-2.2-AA report file (`--format markdown|json` reuse existing renderers; `--out`, `--open`). Score formula (ADR-0008) unchanged. `--serve`/`--format spdx`/hosted are out of scope (ADR-0025).

**Go alignment (binding):** pure renderer over `doctor.Result`; CSS/JS/fonts embedded via `go:embed`; deterministic output (timestamp normalized in tests); `gofumpt`/`golangci-lint`/`gosec`/`go vet`/`-race` clean; ≥85% coverage on `internal/render/html`. Always run the FULL `mise exec -- moon run :lint`.

## Task ordering (build stays green)
`internal/render/html` skeleton + `go:embed` assets → font embedding (subset woff2) → template sections → client JS (filter/search/expand/copy) → `charter report` command → **self-containment + golden tests** → design polish (`.agents/skills/` design set + screenshots) → docs (architecture `report` purpose, mirror) → final review + push.

## Task 1: internal/render/html skeleton
- [ ] New package: `Render(result doctor.Result) ([]byte, error)` producing one HTML document; `go:embed` an `assets/` dir (CSS, JS, template). Inline the `Result` as a JSON payload in a `<script type="application/json">`. Wire nothing to the CLI yet; unit-test it returns a single self-contained document.

## Task 2: fonts (offline embedding)
- [ ] Add Latin-subset `woff2` under `assets/fonts/` (all SIL OFL 1.1 — record license notices): **Ruda** (400/500/700/800 — `--font-site`/`--font-audit`: body/headings/finding-titles/table/badges + the `✦ charter` wordmark), **Atkinson Hyperlegible Mono** (400/500 — `--font-code`, the **primary** mono: CLI/commands/terminal/code/config/inline), **IBM Plex Mono** (400/500 — `--font-meta`, *optional* formal metadata: rule IDs/paths/confidence), **Share Tech** (400 — `--font-accent`, accents only; embed only if used). Embed via base64 `@font-face`; map the `DESIGN-TOKENS.md` `:root` tokens to CSS variables; every stack ends in a system fallback. Keep total embedded payload bounded; if exceeded, fall back to system fonts (no network). **Do not default code/rules to IBM Plex Mono** — Atkinson Hyperlegible Mono is primary.

## Task 3: template sections
- [ ] Build the sections (surpassing the reference): top bar + nav · provenance strip · score hero (numeral, zone label, `role=progressbar` bar + ticks, formula, **cap alert**) · category scorecard grid · findings (auto-fix CTA, severity filter pills + search, expandable cards: Evidence redacted / Remediation / Auto-fix / **Why this matters** = `charter explain` / `helpUri` link, empty state) · suppressions table + governance alert · scan-summary metrics + severity bar · footer. Light/dark via `prefers-color-scheme`.

## Task 4: client interactivity (inlined JS, zero network)
- [ ] Filter by severity/category, text search, expand/collapse (+ all), copy `path:line` with feedback, light/dark. All vanilla JS inlined; **no external fetch**. Keyboard-operable (Enter/Space on cards), `aria-expanded`/`aria-pressed`/`role=progressbar` kept in sync.

## Task 5: charter report command
- [ ] `cmd/charter report`: `--format html` (default) | `markdown` | `json` (reuse `internal/render/{markdown,json}`); `--out <file>` (default `charter-report.html`); `--open` (off by default — print the written path; `--open` launches the browser). Exit codes consistent with `doctor` (report generation = 0; below-threshold semantics documented). Tests for flag matrix.

## Task 6: self-containment + golden tests (load-bearing)
- [ ] Assert the rendered HTML references **no** `http(s)://` loaded asset and no external `src`/`href` resource (anchor `helpUri` links allowed). Golden render of a known `Result` fixture (timestamp normalized). Content assertions: every finding/suppression/category present; evidence redacted; cap notice when capped; empty-findings repo → clean PASS report. `--format markdown|json` byte-match the existing renderers.

## Task 7: design polish + a11y (skills)
- [ ] Apply `frontend-design` + `design-system-patterns` + `visual-design-foundations` + `design-taste-frontend` + `high-end-visual-design` + `ui-ux-pro-max` (tokens/palette/typography/composition) + `interaction-design` (microinteractions/feedback/loading/empty/hover-focus) — respect the tokens. **Screenshot-verify** light + dark. WCAG 2.2 AA pass via `web-design-guidelines` (2.1.1, 2.4.7, 2.5.5/2.5.8, 3.3.2, 4.1.2, 1.3.1). Structural a11y assertions added to the golden.

## Task 8: docs — architecture, mirror
- [ ] Architecture: align `charter report`'s documented purpose to the **HTML report** (html leads; `--format spdx` labeled Phase 1.5); update the §1.8 `report` screen. Regenerate the HTML mirror (CF-2 gate). README current-state.

## Task 9: final review + push
- [ ] `moon run :check` green; dogfood `charter report` on Charter itself → valid self-contained PASS report (open in a browser); self-containment + golden tests green; re-read diff vs ADR-0025 (single-file, no server, no hosted, offline fonts). Push the slice.

## Cross-task invariants
- Self-contained: zero external assets/network; fonts embedded; offline-first (ADR-0001).
- No server (`--serve` deferred), no hosted, no `--format spdx` (Phase 1.5). No raw-source embedding (redacted evidence + `path:line` only).
- Score formula (ADR-0008) unchanged; markdown/json reuse existing renderers. Use the installed `.agents/skills/` design set (see `DESIGN-TOKENS.md`): `frontend-design`, `web-design-guidelines`, `design-system-patterns`, `visual-design-foundations`, `interaction-design`, `design-taste-frontend`, `high-end-visual-design`, `ui-ux-pro-max`, `web-component-design`.
