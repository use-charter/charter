## Task 2: `internal/fix` engine + fixers

**Files:** create `internal/fix/fix.go` (types + registry + Plan + Apply), `internal/fix/diff.go`, `internal/fix/fix_ctx001.go`, `internal/fix/fix_ctx004.go`, `internal/fix/fix_ci002.go`, `internal/fix/fix_test.go`.

- [ ] **Step 1: Types + registry + Plan** (`fix.go`):
  ```go
  type Action int
  const ( Create Action = iota; Append )
  type FilePlan struct { RuleID, Path string; Action Action; NewContents []byte; Diff string }
  type Options struct { Rule string } // empty = all fixable

  // fixer returns a plan for a rule, ok=false when nothing to do.
  type fixer func(root string, inv repository.Inventory) (FilePlan, bool, error)
  var registry = map[string]fixer{
  	"AE-CTX-001": fixCTX001,
  	"AE-CTX-004": fixCTX004,
  	"AE-CI-002":  fixCI002,
  }
  // Fixable reports whether a rule has a registered fixer (CLI uses it to flag manual-only rules).
  func Fixable(ruleID string) bool { _, ok := registry[ruleID]; return ok }

  func Plan(result doctor.Result, root string, inv repository.Inventory, opts Options) ([]FilePlan, error) {
  	seen := map[string]bool{}
  	var plans []FilePlan
  	for _, f := range result.Findings {
  		if opts.Rule != "" && f.RuleID != opts.Rule { continue }
  		fx, ok := registry[f.RuleID]
  		if !ok || seen[f.RuleID] { continue }
  		seen[f.RuleID] = true
  		plan, ok, err := fx(root, inv)
  		if err != nil { return nil, err }
  		if ok { plans = append(plans, plan) }
  	}
  	return plans, nil
  }
  ```
- [ ] **Step 2: Unified diff** (`diff.go`): `buildCreateDiff(path string, contents []byte) string` → `--- /dev/null\n+++ b/<path>\n@@ -0,0 +1,N @@\n` + each content line prefixed `+`. `buildAppendDiff(path string, existingTail, added []string) string` → a hunk showing up to 3 trailing context lines of `existingTail` (space-prefixed) then `added` lines (`+`-prefixed) with a correct `@@` header. Deterministic; trailing-newline-safe.
- [ ] **Step 3: Fixers:**
  - `fix_ctx001.go`: `fixCTX001(root, inv)` — if any agent context file exists (reuse `agentcontext.Files` / inv), return `ok=false`; else build `scaffold.AGENTSMarkdown(scaffold.Detect(root))`, return a `Create` plan for `AGENTS.md` with `buildCreateDiff`.
  - `fix_ctx004.go`: compute missing patterns vs the current `.gitignore` (read via inv/os). If `.gitignore` absent → `Create` (`scaffold.Gitignore()`). Else if any of `.charter/`,`*.charter-session`,`.claude/local/`,`.cursor/cache/`,`.hk/`,`.env*` is missing → `Append` plan whose `NewContents` is the missing lines (prefixed with a blank line + `# Charter / agent session artifacts` comment) and `Diff` via `buildAppendDiff`. Else `ok=false`.
  - `fix_ci002.go`: if any tracked workflow already provides the charter-gate (`charter doctor` text or `use-charter/charter-action@`), `ok=false`; else `Create` `.github/workflows/charter.yaml` with the exact template from the design spec (checkout SHA-pinned — resolve/keep the SHA used elsewhere in the repo, e.g. `actions/checkout` SHA from `.github/workflows/ci.yml`; `use-charter/charter-action@v1`).
- [ ] **Step 4: Applier** (`fix.go`): `Apply(root string, plans []FilePlan) ([]string, error)` — for each plan: `abs := filepath.Join(root, filepath.FromSlash(plan.Path))`; if `abs` exists, copy it to `filepath.Join(root, ".charter", "backups", ts, plan.Path)` (MkdirAll parents, `ts` a fixed RFC3339-ish stamp computed once per Apply); then for `Create` write `NewContents` (only if absent — re-stat), for `Append` read existing + append `NewContents` and write back. Never `os.Remove`/truncate-unrelated; `0o644` files, `0o755` dirs; gosec-annotate. Return the list of written paths + the backup dir.
- [ ] **Step 5: Tests** (`fix_test.go`): registry has exactly the 3 rules + `Fixable("AE-SEC-001")==false`; `Plan` produces correct `FilePlan`s (golden diffs); `fixCTX004` appends only missing patterns; `Apply` backs up an existing `.gitignore` to `.charter/backups/...` and appends (assert the backup matches the original bytes and the new file = original+missing); `Create` writes only to an absent path; `Apply` never touches an unrelated file (snapshot a sibling file before/after).

## Decisions (locked in ADR-0020)

1. `internal/fix`: pure planner + diff; applier backs up to `.charter/backups/<ts>/` before writing, never deletes/truncates, never targets secret/dangerous rules.
2. Fixers: AE-CTX-001 (create AGENTS.md), AE-CTX-004 (append/create .gitignore), AE-CI-002 (create charter.yaml using use-charter/charter-action@v1).
3. AE-CI-002 generalized to recognize direct/action CI forms (test/build, actionlint+zizmor, security scanners); first-party `use-charter/charter-action@` exempt from the pin check. Charter's moon CI still passes.
4. CLI `fix --rule/--dry-run/--all/--yes/--path`; exit 0/2.
5. Armed-not-fired: scaffolded charter.yaml references the (launch-published) action; statically clears AE-CI-002 now.

A change to any of these during execution updates the ADR (Task 5 Step 1).
