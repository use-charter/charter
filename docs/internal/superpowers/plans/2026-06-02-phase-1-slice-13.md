# Phase 1 Slice 13 Implementation Plan — MCP Catalog v1

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Derived from:** `docs/internal/superpowers/specs/2026-06-02-phase-1-slice-13-design.md` (design spec) and `docs/internal/decisions/0021-mcp-catalog-v1.md` (ADR-0021). The spec/ADR own the *what/why*; this plan owns the *how*. If they disagree, the spec/ADR win.

**Branch & commit/push policy:** work **directly on `main`**. Commit after each task (orchestrator commits after the two-stage review). **Push the whole slice together** once `moon run :check` is green (final task). GPG-signed.

**Goal:** ship the embedded, founder-curated MCP Catalog (`internal/catalog`) and make `AE-MCP-001` catalog-aware (deprecated → HIGH, advisory → HIGH, behind-stable → informational, one finding/server) and `AE-MCP-002` recognize a vendor trusted-host baseline. Exact-match comparison only (staleness-safe). Implements ADR-0021.

**Tech stack:** Go 1.26.3, `gopkg.in/yaml.v3` (already a dep), `//go:embed`; `internal/rules/mcp`, `internal/findings`, `internal/scoring`, `internal/rules/catalog`. Module `go.use-charter.dev/charter`. No new dependency; no network; no LLM.

**Go alignment (binding):** catalog lookups are **pure**; `Parse` wraps errors `%w`; `Default()` parses the embed once via `sync.Once`; no cross-scheme version ordering (string equality against curated sets only); no `_` discards; deterministic ordering preserved; `gofumpt`/`golangci-lint`/`gosec`/`go vet`/`-race` clean; ≥85% coverage for `internal/catalog`. **Always run the FULL `mise exec -- moon run :lint`**, not just `go vet`/`gofumpt -l`.

- [ ] **Step 2: Types + loader + lookups** in `catalog.go` (package doc comment explains the exact-match / staleness-safe contract and that the embed is build-validated):
  - `//go:embed data/catalog.yaml` into a `[]byte`; `Default() *Catalog` parses once with `sync.Once` (on parse error, `panic` with a wrapped message — unreachable in shipped builds because the validity test gates it).
  - `Parse(data []byte) (*Catalog, error)` (yaml.v3 `UnmarshalStrict`-style via `yaml.Unmarshal`; wrap `%w`). YAML keys are camelCase via struct tags (`yaml:"stableVersion"` etc.).
  - Types exactly as the spec (`Catalog`, `ServerEntry`, `Advisory`) with yaml tags.
  - `(*Catalog) Lookup(pkg string) (ServerEntry, bool)` — build a `map[string]ServerEntry` once (store on the struct at parse time, or linear scan; linear is fine for ~15 entries — keep it simple + pure).
  - `(ServerEntry) AdvisoryFor(version string) (Advisory, bool)` — first advisory whose `Affected` contains `version` exactly.
  - `(ServerEntry) KnownBehind(version string) (string, bool)` — returns `(StableVersion, true)` iff `version` is in `KnownVersions`, `version != StableVersion`, **and** `AdvisoryFor(version)` is false; else `("", false)`.
  - `(*Catalog) TrustedHostSet() map[string]struct{}` — lowercased, trimmed hosts.
- [ ] **Step 3: Validity test** (`catalog_test.go`, `TestCatalogValid` on `catalog.Default()`) — the curation contract:
  - non-empty `Version`/`Generated`; ≥1 server; ≥1 trusted host.
  - unique `Package`; `Ecosystem ∈ {npm,pypi}`; `Status ∈ {active,deprecated}`.
  - `TrustedHosts`: all equal to their lowercased-trimmed form; no duplicates.
  - `deprecated` → `Successor != ""`, `StableVersion == ""`, `KnownVersions` empty.
  - `active` → if `KnownVersions` non-empty: ascending + unique, and `StableVersion == KnownVersions[len-1]`; if `KnownVersions` empty: `StableVersion == ""`.
  - every `Advisory` (none in the seed today, but enforce): `ID != ""`, `len(Affected) > 0`, `FixedIn != ""`, `Severity == "high"`.
- [ ] **Step 4: Lookup units** (inline `Parse` of a tiny YAML): `Lookup` hit/miss; `AdvisoryFor` exact hit + miss (unaffected + newer version); `KnownBehind` — in-list non-stable → behind; stable → not; absent → not; advisory-covered version → not behind (advisory precedence); `TrustedHostSet` lowercases mixed-case input.

## Task 2: `AE-MCP-001` catalog-aware precedence ladder

**Files:** modify `internal/rules/mcp/mcp001.go`, `internal/rules/mcp/mcp001_test.go`.

- [ ] **Step 1:** Change `checkPinning(files []ConfigFile)` → `checkPinning(files []ConfigFile, cat *catalog.Catalog)`. Inside the per-server loop, after resolving `token` and `name, version, pinned := classifyPackageSpec(token)`, apply the ladder (emit at most ONE finding per server, then `continue`):
  1. `entry, ok := cat.Lookup(name)`.
  2. `ok && entry.Status == "deprecated"` → HIGH, summary `"MCP server package <name> is archived/deprecated — migrate to <successor> (supply-chain maintenance risk)"`, remediation references the successor, evidence `cf.Path + ": server " + s.Name + " uses " + token + " (archived)"`.
  3. `!pinned` → existing unpinned HIGH (unchanged).
  4. pinned + `ok`: `adv, hit := entry.AdvisoryFor(version)` → HIGH, summary `"MCP server <name>@<version> is affected by <adv.ID> — fixed in <adv.FixedIn>"`, remediation `"Upgrade <name> to <adv.FixedIn> or later"`, evidence includes `adv.Summary`/`adv.Reference`.
  5. pinned + `ok`: `stable, behind := entry.KnownBehind(version)` → **informational** (`Informational: true`), `Severity: findings.SeverityLow` is **wrong** — keep it out of scoring via `Informational` and set a representative severity that won't deduct; follow the `AE-SUPPRESS-003` shape (it uses `SeverityMedium` + `Informational:true`). Summary `"MCP server <name> is pinned to <version>; catalog stable version is <stable> — upgrade available"`, remediation `"Bump <name> to <stable> when convenient"`.
  6. else clean.
  Keep `RuleID "AE-MCP-001"`, `Category "MCP Safety"`, `Locations[0] = {cf.Path, s.Line}`, and the existing deterministic sort.
- [ ] **Step 2:** Update the two existing `checkPinning(...)` call sites in `mcp001_test.go` to pass a catalog. Use an **inline** catalog via `catalog.Parse` for facet tests; for the existing `TestCheckPinningFlagsLatest`/`TestCheckPinningAllPinned`, pass `catalog.Default()` (the `1.0.4` pins are absent from `knownVersions` → still clean; `gumroad-mcp` not in catalog → still unpinned HIGH). Confirm both still assert the same counts.
- [ ] **Step 3: Facet tests** (`mcp001_test.go`, inline catalogs via `catalog.Parse`):
  - **deprecated:** a server pinned to `@modelcontextprotocol/server-github@1.2.3` (inline catalog marks it deprecated → successor `X`) → 1 HIGH, message contains `archived`/the successor. Also `@latest` of a deprecated package → still the deprecated HIGH (precedence over unpinned).
  - **advisory:** inline catalog with `server-foo` advisory `{id: GHSA-xxxx, affected: ["1.0.0"], fixedIn: "1.0.1", severity: high}`; pin `foo@1.0.0` → HIGH with `GHSA-xxxx`/`1.0.1`; pin `foo@1.0.1` → clean.
  - **behind-stable:** inline catalog `server-bar` `knownVersions: ["1.0.0","1.0.1"]`, `stableVersion "1.0.1"`; pin `bar@1.0.0` → **informational** (`Informational==true`, RuleID AE-MCP-001); pin `bar@1.0.1` → clean; pin `bar@9.9.9` (absent) → clean (staleness-safe).
  - **scoring:** assert a behind-stable finding does **not** change the score (reuse `internal/scoring`: a result with only the informational finding scores 100).
