## Task ordering (build stays green)

Language-detection helper → AE-CTX-006 (smallest) → AE-TEST-001 → AE-AUTO-001 → scorecard → catalog/specs/audit/architecture/roadmap docs → FP-validation + tune → final review + push.

## Task 1: language detection helper
- [ ] A small inventory-based language detector (reuse the `scaffold.Detect` manifest patterns) usable by the testing + autonomy rules. Decide: shared helper in a new `internal/langdetect` (pure, over `repository.Inventory`) or per-package. Detect Go/JS-TS/Python/Rust at minimum (+ Java/Kotlin/Ruby/C#/PHP for AE-TEST). Unit-test detection.

## Task 2: AE-CTX-006 (context emphatic density)
- [ ] In `internal/rules/context`, add `AE-CTX-006`: count emphatic tokens / 1K words in the agent context file; `Informational: true`; threshold const (15/1K, tunable). Depends on a context file existing. Unit tests (over/under; informational non-deducting via scoring assertion). Wire into the context rule entrypoint.

## Task 3: AE-TEST-001 (tests present)
- [ ] New `internal/rules/testing`: for each detected language, require ≥1 test artifact (table in the spec). One High finding per language-without-tests; N/A when no recognized language. Wire into `doctor.Run`. Unit + fixture tests (Go-with/without tests; JS; Python; non-code N/A; multi-language).

## Task 4: AE-AUTO-001 (verification command discoverable)
- [ ] New `internal/rules/autonomy`: pass when a task runner declares a test entrypoint (Makefile/justfile/Taskfile/package.json scripts/mise/moon) OR a conventional toolchain applies (Go/Rust). Medium finding when a recognized language exists but neither path is discoverable; N/A for non-code. Wire into `doctor.Run`. Unit matrix tests (runner target; conventional toolchain; JS-with-tests-no-script fires; Charter's moon passes).

## Task 5: category scorecard (reporting)
- [ ] `internal/render/json` DTO: add `categories: [{category, findings, deduction, worst_severity}]` (additive). `internal/render/{text,markdown}`: add a "Readiness by category" block. Score formula untouched. Regenerate the SARIF golden only if affected (it shouldn't be). Tests for the breakdown.

## Task 6: docs — catalog, specs, audit, architecture, roadmap
- [ ] `internal/rules/catalog`: add `AE-TEST-001`/`AE-AUTO-001`/`AE-CTX-006` (+ catalog drift test passes). New specs `docs/internal/specs/AE-{TEST,AUTO}-001.md`, `AE-CTX-006.md`. Audit checklist entries. Architecture §0.2 rule table (15→18) + a short milestone note. Renumber `2026-06-01-v1-launch-roadmap.md` (14→15…19→20; insert new Slice 14). Regenerate HTML mirror (CF-2 `docs-html` gate).

## Task 7: FP-validation + tune
- [ ] Scan ≥10 real public repos (Go/JS/Python/Rust/polyglot); classify every AE-TEST-001/AE-AUTO-001 finding TP/FP; target ≤10% FP; tune detection/threshold; record results.

## Task 8: final review + push
- [ ] `moon run :check` green; dogfood `charter doctor --path .` = **100**; re-read diff vs ADR-0023 (severities, FP guards, formula unchanged, no external refs); push the slice.

## Cross-task invariants
- Score formula (ADR-0008) unchanged; scorecard is presentation only.
- FP discipline (#9): TEST N/A for non-code; AUTO counts conventional toolchains; CTX-006 never deducts.
- No network, no LLM, no new runtime dependency. No external references anywhere.
