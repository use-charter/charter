## Task 1: `AE-CTX-002` verification-marker fix (`internal/rules/context`)

**Files:** modify `internal/rules/context/ctx001.go`, `internal/rules/context/ctx002.go`, `internal/rules/context/context_test.go`; modify `docs/internal/specs/AE-CTX-002.md` and `docs/internal/audit/charter-v1-audit-checklist.md` (AE-CTX-002 note).

- [ ] **Step 1: Extract a shared verification-signal helper.** In `ctx001.go`, the verification signal is currently inline in `missingContextSignals`: `hasAnySignal(lower, "moon run :check", "charter doctor", "verification command", "verify with")`. Extract it:
```go
// hasVerificationSignal reports whether the context references a recognized
// verification command. Shared by AE-CTX-001 (must have one) and AE-CTX-002
// (the stated command must be a recognized one — not hardcoded to this repo's
// `moon run :check`).
func hasVerificationSignal(lower string) bool {
	return hasAnySignal(lower, "moon run :check", "charter doctor", "verification command", "verify with")
}
```
Use it in `missingContextSignals` (replace the inline call).

- [ ] **Step 2: Generalize `ctx002.go`.** In `missingRepoTruthMarkers`, REMOVE the hardcoded `"moon run :check"` from `requiredMarkers`. Instead, after the literal-marker loop (which still checks `.env*`, `secrets/`, and the conditional `hk.pkl`/arch-doc), add a verification check using the shared helper:
```go
func missingRepoTruthMarkers(content string, inv repository.Inventory) []string {
	var missing []string
	lower := strings.ToLower(content)

	if !hasVerificationSignal(lower) {
		missing = append(missing, "verification command")
	}

	requiredMarkers := []string{".env*", "secrets/"}
	if inv.Has("hk.pkl") {
		requiredMarkers = append(requiredMarkers, "hk.pkl")
	}
	if inv.Has("docs/internal/architecture/charter-architecture-2026.md") {
		requiredMarkers = append(requiredMarkers, "docs/internal/architecture/charter-architecture-2026.md")
	}
	for _, marker := range requiredMarkers {
		if !strings.Contains(content, marker) {
			missing = append(missing, marker)
		}
	}
	return missing
}
```
(Keep the rest of `checkCTX002` unchanged. The evidence list now says `verification command` instead of `moon run :check` when absent.)

- [ ] **Step 3: Tests** in `context_test.go`: add cases asserting (a) an `AGENTS.md` with `charter doctor` + `.env*` + `secrets/` (no `moon run :check`) does NOT fire AE-CTX-002; (b) an `AGENTS.md` lacking any verification signal DOES fire it (evidence contains `verification command`); (c) Charter's own `moon run :check` style still passes. Run the failing case first if practical.

- [ ] **Step 4: Docs.** `docs/internal/specs/AE-CTX-002.md` + the audit checklist AE-CTX-002 entry: where they imply `moon run :check`, generalize to "a recognized verification command (e.g. `charter doctor`, `moon run :check`)". Keep edits terse.

- [ ] **Step 5: Verify** `mise exec -- go test ./internal/rules/context -race -count=1 -v`; `mise exec -- go run ./cmd/charter doctor --path . --threshold 80` still **100** (Charter's own AGENTS.md still matches); `go vet` + `gofumpt -l internal/rules/context` clean; `mise exec -- moon run :docs`. Commit (orchestrator): `fix: accept any recognized verification command in AE-CTX-002`.

## Task 2: `internal/scaffold` engine (detection + templates + plan)

**Files:** create `internal/scaffold/detect.go`, `internal/scaffold/templates.go`, `internal/scaffold/plan.go`, `internal/scaffold/scaffold_test.go`.

- [ ] **Step 1: Detection** (`detect.go`). `Detect(root string) (Project, error)` reading tracked manifests (pure file reads; no git shell needed — read files directly under root):
```go
type Lang struct{ Name, Version string }
type Project struct {
	Name     string   // repo dir base name
	Langs    []Lang   // detected, stable order
	CI       string   // "GitHub Actions" or ""
	Agents   []string // "claude","cursor","copilot" detected
}
```
Detect: `go.mod` → Go + version from the `go`/`toolchain` line; `package.json` → JS/Node (+ engines if present); `pyproject.toml` → Python; `Cargo.toml` → Rust. CI: `.github/workflows/` has any `*.yml`. Agents: `.claude/` dir, `.cursor/` dir, `.github/copilot-instructions.md`. Deterministic, offline.

- [ ] **Step 2: Templates** (`templates.go`) — pure builders returning `[]byte`. Implement `AGENTSMarkdown(p Project) []byte` producing exactly the structure in the design spec (Project Overview, Tech Stack from `p.Langs`+CI, Commands with `charter doctor` verification + best-effort build/test per primary language, Edit Boundaries with `.github/workflows/`/`.env*`/`secrets/`/`db/migrations/`/`terraform/`, Verification). Keep it < 600 est. tokens (assert in test). Also:
  - `CharterYAML(profile string) []byte` → `policy:\n  profile: <profile>\n`.
  - `Gitignore() []byte` → a `# Charter / agent artifacts` block: `.charter/`, `*.charter-session`, `.claude/local/`, `.cursor/cache/`.
  - `ClaudeSettings() []byte` → `{ "$schema": "https://json.schemastore.org/claude-code-settings.json", "permissions": { "deny": ["Read(./.env)","Read(./.env.*)","Read(./secrets/**)"] } }` (valid JSON, MCP-free).
  - `ArchitectureMarkdown(p Project) []byte`, `EnvExample() []byte` → minimal useful templates.

- [ ] **Step 3: File plan** (`plan.go`).
```go
type Action int
const ( Create Action = iota; Skip )
type FileAction struct { Path string; Action Action; Contents []byte }
type Options struct { Profile string; Agents []string }
// Plan computes the actions given the project, options, and a presence predicate
// (exists(relPath) bool) — injected so it's unit-testable without disk.
func Plan(p Project, opts Options, exists func(string) bool) []FileAction
```
Candidate files (stable order): `AGENTS.md`, `charter.yaml`, `.gitignore`, `ARCHITECTURE.md`, `.env.example`, and `.claude/settings.json` iff `claude` ∈ opts.Agents. Each → `Create` with contents if `!exists(path)`, else `Skip`.

- [ ] **Step 4: Tests** (`scaffold_test.go`): detection (Go/JS/Python/none, CI present/absent, agents); `AGENTSMarkdown` contains all CTX-001 signals + the CC-002 off-limits tokens and is < 600 tokens (use the same `(len+3)/4` estimate); `ClaudeSettings` is valid JSON with the secret denies and no `mcp` key; `Plan` marks existing files `Skip` and absent ones `Create`, and includes `.claude/settings.json` only when claude is requested.

- [ ] **Step 5: Verify** `mise exec -- go test ./internal/scaffold -race -count=1 -v -cover` (≥ 85%); `go vet` + `gofumpt -l internal/scaffold` clean. Commit (orchestrator): `feat: add internal/scaffold engine for charter init`.

## Task 3: `cmd/charter/init.go` (command)

**Files:** create `cmd/charter/init.go`, `cmd/charter/init_test.go`; modify `cmd/charter/root.go`.

- [ ] **Step 1: Command** (model after `suppress.go`). `newInitCommand()`:
  - Flags: `--path` (""), `--profile` ("standard"), `--agents` (csv, ""), `--dry-run` (false), `--yes` (false, accepted/no-op).
  - Validate `--profile` ∈ {strict,standard,relaxed} else exit 2.
  - Resolve target: try `repository.ResolveRoot(path)`; if that errors (no `.git`), fall back to the literal `--path` or CWD (so `init` works on a not-yet-git dir). 
  - `agents`: if `--agents` set, parse csv; else use `scaffold.Detect(...).Agents`; if still empty, default `["claude"]`.
  - `p, _ := scaffold.Detect(root)`; `actions := scaffold.Plan(p, scaffold.Options{Profile: profile, Agents: agents}, func(rel string) bool { _, err := os.Stat(filepath.Join(root, filepath.FromSlash(rel))); return err == nil })`.
  - Apply: for `Create`, `os.MkdirAll(parent, 0o755)` then **re-check absence** (`os.Stat`) and `os.WriteFile(0o644)` only if absent (TOCTOU-safe; never overwrite); for `Skip`, count it. `--dry-run` → print plan, write nothing. **Never** remove/truncate.
  - Report to stdout: per-file `create`/`skip` lines, a `created N · skipped M` summary, and `› Next: charter doctor`. Exit 0; IO/flag errors → `commandExitError{exitCode:2}`.
- [ ] **Step 2: Register** in `root.go`: `cmd.AddCommand(newInitCommand())`.
- [ ] **Step 3: CLI tests** (`init_test.go`, model after `doctor_test.go`/`suppress_test.go` using `newRootCommand()` + `t.TempDir()`): init on a blank temp Go repo creates the expected files; re-run reports all `skip` and does NOT change file contents (capture before/after bytes — proves zero overwrite); `--dry-run` writes nothing; `--profile strict` → `charter.yaml` contains `profile: strict`; invalid profile → exit 2; `--agents ""`/none still creates the core set.
- [ ] **Step 4: Verify** `mise exec -- go test ./cmd/charter -race -count=1 -v`; `go build ./...`; `go vet`; `gofumpt -l cmd/charter` clean; dogfood `mise exec -- go run ./cmd/charter init --dry-run --path .` on Charter's repo → reports all `skip` (everything already exists; zero creates). Commit (orchestrator): `feat: add charter init command`.

## Decisions (locked in ADR-0019)

1. Create-missing-only; never overwrite/delete (re-check before write; no Remove/truncate). `--dry-run` previews. Non-interactive (`--yes` implicit).
2. Scaffold set: AGENTS.md (agents.md standard, passes CTX-001/CC-002, ≤600 tokens), charter.yaml (profile), .gitignore (agent artifacts), ARCHITECTURE.md, .env.example, .claude/settings.json (MCP-free, secret-deny) when claude. AGENTS.md is the universal context (no .cursor duplication).
3. Deterministic offline detection (Go robust; others best-effort) populates AGENTS.md.
4. Flags `--path/--profile/--agents/--dry-run/--yes`; exit 0 success, 2 error.
5. AE-CTX-002 accepts any recognized verification command (shared helper with AE-CTX-001), not only `moon run :check` — FP fix.
6. ≥80 OOTB proven by an init→doctor integration test on a synthesized blank Go repo.

A change to any of these during execution updates the ADR (Task 5 Step 1), not just the plan.
