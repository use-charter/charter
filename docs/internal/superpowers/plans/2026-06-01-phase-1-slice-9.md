**Derived from:** `docs/internal/superpowers/specs/2026-06-01-phase-1-slice-9-design.md` (design spec), `docs/internal/decisions/0016-release-pipeline-and-supply-chain.md` (ADR-0016), `docs/internal/decisions/0017-version-command-and-build-stamping.md` (ADR-0017). **Branch & commit/push policy:** work **directly on `main`** (no feature branch). Commit after each task. **Do not push per task — push the whole slice's commits to `main` together once `moon run :check` is green** (Task 5). Commits are GPG-signed via macOS pinentry/Keychain; `git commit` works non-interactively.

**Goal:** Wire `charter version` and Charter's signed release rails (GoReleaser v2 → binaries + `checksums.txt` + syft SPDX SBOMs + cosign keyless bundle + SLSA L3 provenance → GitHub Release + Homebrew cask). Implements ADR-0016/0017. Unblocks Slice 10 (the Action downloads the released binary).

**Armed, not fired:** this slice builds and validates everything verifiable **without** cutting a tag or writing to Rekor — `goreleaser check`, an offline `--snapshot` build, `charter version`, actionlint/zizmor, `moon run :check`. The first real signed release (`v0.9.0`) is a go-public ops action. **Do not `git tag`/push a tag in this slice.**

**Tech stack:** Go 1.26.3, Cobra, `internal/version`; GoReleaser v2.16.0, cosign v3.0.6, syft (pin latest), slsa-github-generator v2.1.0; mise (aqua) + Moon; `runtime`/`runtime/debug` (stdlib). Module path `go.use-charter.dev/charter`; remote `github.com/tashfiqul-islam/charter`.

**Go alignment (binding):** `internal/version` stays pure (ldflags vars are write-once/read-only; deterministic `"unknown"` fallback via `runtime/debug`); the `version` command is pure-stdout, exit 0; `gofumpt`/`go vet`/`-race` clean; no `_` discards; no new runtime dependencies.

- [ ] **Step 1: Extend `internal/version/version.go`** — add `commit`/`date` ldflags vars and accessors (keep `Version()` as-is):

```go
package version

import "runtime/debug"

// injected, commit, and date are overridden at release time via
// -ldflags "-X go.use-charter.dev/charter/internal/version.<name>=<value>"
// (ADR-0016 GoReleaser injects Version/FullCommit/CommitDate).
var (
	injected string
	commit   string
	date     string
)

// Version returns the Charter build version: the release value injected via
// ldflags when set, else the module version embedded by the Go toolchain (e.g.
// for `go install go.use-charter.dev/charter/cmd/charter@v1.2.3`), else a
// development placeholder.
func Version() string {
	if injected != "" {
		return injected
	}
	if info, ok := debug.ReadBuildInfo(); ok {
		if v := info.Main.Version; v != "" && v != "(devel)" {
			return v
		}
	}
	return "0.0.0-dev"
}

// Commit returns the VCS revision: the ldflags-injected value, else the
// toolchain-embedded vcs.revision, else "unknown".
func Commit() string {
	if commit != "" {
		return commit
	}
	return buildSetting("vcs.revision")
}

// Date returns the build/commit timestamp: the ldflags-injected value, else the
// toolchain-embedded vcs.time, else "unknown".
func Date() string {
	if date != "" {
		return date
	}
	return buildSetting("vcs.time")
}

func buildSetting(key string) string {
	if info, ok := debug.ReadBuildInfo(); ok {
		for _, s := range info.Settings {
			if s.Key == key && s.Value != "" {
				return s.Value
			}
		}
	}
	return "unknown"
}
```

## Task 3: `.github/workflows/release.yml` rewrite (GoReleaser + SLSA provenance)

**Files:** modify `.github/workflows/release.yml`.

- [ ] **Step 1: Replace `release.yml`** with the two-job pipeline (reuse the exact `actions/checkout` + `jdx/mise-action` SHAs already pinned in `ci.yml`):

```yaml
name: Release

on:
  push:
    tags:
      - "v*"

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read

jobs:
  goreleaser:
    name: Build, sign, and release
    runs-on: ubuntu-latest
    permissions:
      contents: write
      id-token: write
    outputs:
      hashes: ${{ steps.hash.outputs.hashes }}
    steps:
      - name: Check out repository
        uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6.0.2
        with:
          persist-credentials: false
          fetch-depth: 0
      - name: Set up pinned toolchain
        uses: jdx/mise-action@1648a7812b9aeae629881980618f079932869151 # v4
        with:
          version: 2026.5.15
          install: true
          install_args: --locked
          cache: true
      - name: Run GoReleaser
        run: mise x -- moon run :release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          TAP_OWNER: ${{ github.repository_owner }}
          HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}
      - name: Compute SLSA subject hashes
        id: hash
        run: echo "hashes=$(base64 -w0 < dist/checksums.txt)" >> "$GITHUB_OUTPUT"

  provenance:
    name: SLSA provenance
    needs: goreleaser
    permissions:
      actions: read
      id-token: write
      contents: write
    # zizmor: ignore[unpinned-uses] slsa-verifier resolves the builder identity
    # from this semantic tag; SHA-pinning is unsupported here (ADR-0016).
    uses: slsa-framework/slsa-github-generator/.github/workflows/generator_generic_slsa3.yml@v2.1.0
    with:
      base64-subjects: ${{ needs.goreleaser.outputs.hashes }}
      upload-assets: true
```

## Decisions (locked in ADR-0016 / ADR-0017)

1. GoReleaser v2 pinned in mise + driven by Moon (`release`/`release-snapshot`/`release-check`; `release-check` in `:check`); no goreleaser-action/cosign-installer.
2. Reproducible builds (`-trimpath`, `mod_timestamp`); ldflags inject version/commit/date into `internal/version`.
3. sha256 `checksums.txt`; syft SPDX SBOMs (emit syft's actual version; reconcile the doc).
4. cosign v3 keyless **bundle** (`sign-blob --bundle …sigstore.json`) over the checksum file; verify via `verify-blob --bundle … --certificate-identity … --certificate-oidc-issuer …`.
5. SLSA L3 via `generator_generic_slsa3.yml@v2.1.0` (tag-pinned — the single zizmor `unpinned-uses` exception, justified by ADR-0016).
6. Distribution: signed GitHub Release + `homebrew_casks` (quarantine-strip hook); `go install` deferred (vanity redirect).
7. Parametric owner (`github.repository*`, `TAP_OWNER`); no hardcoded org.
8. Armed, not fired: validate offline (`check` + `--snapshot`); first real signed tag is go-public ops. **No tag in this slice.**

A change to any of these during execution requires updating the ADR (Task 4 Step 1), not just the plan.

## Self-review

- **Spec coverage:** version command → T1; GoReleaser config + tooling + Moon tasks → T2; signed workflow + SLSA → T3; docs → T4; verification + push → T5.
- **Build-green ordering:** T1 is pure Go and self-contained; T2 adds config/tooling validated offline (no code import changes); T3 only touches the workflow; T4 docs; T5 verifies.
- **Error contract:** `version` command can't fail (pure stdout, exit 0); release failures surface from goreleaser/cosign/slsa with non-zero exits in CI.
- **No placeholders that block:** all Go is complete/compilable; YAML blocks are complete but `goreleaser check` (T2.4) + actionlint/zizmor (T3.2) are the authoritative validators and any v2 schema drift is fixed there. The only deliberate "verify-then-pin" is syft's exact version (T2.1) and SPDX version (T2.4/T4.5).
- **Safety:** no tag pushed; `dist/` git-ignored; no keys/secrets committed; OIDC permissions job-scoped; top-level `contents: read`.
