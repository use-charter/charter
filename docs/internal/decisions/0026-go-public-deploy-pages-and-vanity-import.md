# ADR-0026 Go-Public Deploy — Cloudflare Pages Landing + Vanity-Import Worker

- Status: Accepted
- Context: Slice 19 ships the public landing site, and going public requires the live edge that earlier slices deferred. Two carry-forward items converge here: the landing site needs a host, and **CF-4** — `go install go.use-charter.dev/charter/cmd/charter@…` cannot resolve until `go.use-charter.dev` serves a `go-import` meta tag (ADR-0016 deferred the "`go install` vanity redirect"; carry-forward CF-4 assigned it to "Slice 18/19 web build + go-public DNS/deploy"). The module path has been the **vanity** `go.use-charter.dev/charter` since Slice 1 while the repo lives at `github.com/use-charter/charter` (ADR-0016, ADR-0002 single module), precisely so the import path is decoupled from the VCS host — but that only pays off once the vanity host is live. Until now install worked **only** via signed binaries + Homebrew cask (ADR-0016); `go install` was the visible gap. Verified live against official sources (2026-06-12, ADR-0006): Cloudflare still treats **Pages** as first-class (`pages_build_output_dir` in `wrangler.toml`/`wrangler.jsonc`, auto-detected `functions/`), while steering *new* projects toward **Workers Static Assets** ([migrate-from-pages](https://developers.cloudflare.com/workers/static-assets/migration-guides/migrate-from-pages/)); Workers **routes** bind a path pattern on an existing proxied zone record, **Custom Domains** (`custom_domain = true`) auto-provision a hostname + TLS with no DNS record ([routing](https://developers.cloudflare.com/workers/configuration/routing/), [wrangler config](https://developers.cloudflare.com/workers/wrangler/configuration/)); and the Go tool resolves a vanity path by fetching `https://<path>?go-get=1` over HTTPS and reading `<meta name="go-import" content="<module-prefix> <vcs> <repo-url>">` ([cmd/go remote import paths](https://pkg.go.dev/cmd/go#hdr-Remote_import_paths), [go.dev/ref/mod#vcs-find](https://go.dev/ref/mod#vcs-find)). The landing site is already built and tested as a static Astro build with a Pages Function at `/api/waitlist` that calls Resend (the `send_email` binding is Workers-only, unavailable to Pages Functions — established during Slice 19). Config to date lived only as prose in `docs/product/DEPLOY.md`; there was no committed, reproducible deploy artifact for either host.
- Decision: Charter goes public as **one Cloudflare account, two committed Workers, one Pages project** — infrastructure-as-code under a new root `infra/`, all `wrangler.toml`.
  1. **Landing = Cloudflare Pages, not Workers Static Assets.** The site keeps its shipped, tested shape: static Astro (`output: 'static'`, no adapter), auto-detected `web/functions/api/waitlist.ts`, `web/public/_headers` for CSP/HSTS. Config is captured in `web/wrangler.toml` (`name = "charter-landing"`, `pages_build_output_dir = "./dist"`, `compatibility_date`); build command/root and the `RESEND_API_KEY` secret + `WAITLIST_TO` var stay dashboard-set. We **reject** migrating to Workers Static Assets for v1: it would rewrite the verified Pages Function into a Worker handler with an `assets` binding + `run_worker_first` for `/api/*` — real churn on tested code for no launch-blocking gain. Pages remains fully supported; the migration is recorded as a **future option**, not a v1 task.
  2. **`charter-router` (`infra/router/`) fronts `use-charter.dev/*`.** A catch-all route (`pattern = "use-charter.dev/*"`, `zone_name`) where the worker, not DNS, owns path logic: `/docs/*` and `/rules/*` proxy to Mintlify (`MINTLIFY_ORIGIN`, default `charter.mintlify.app`) with the public host forwarded; `/.well-known/acme-challenge/` resolves at the edge; everything else re-bases the request onto the Pages origin (`LANDING_ORIGIN`, e.g. `charter-landing.pages.dev`), preserving method + body so `/api/waitlist` and every asset flow through. `LANDING_ORIGIN` is dashboard-set so no placeholder ships by accident. The apex needs a proxied (orange-cloud) record for the route to bind.
  3. **`charter-go-vanity` (`infra/go-vanity/`) closes CF-4.** A Worker on `go.use-charter.dev` (`custom_domain = true`, auto hostname + cert) answers `?go-get=1` with the `go-import` (and `go-source`) meta for module root `go.use-charter.dev/charter` → `git https://github.com/use-charter/charter`, and redirects human browsers to `pkg.go.dev` (subpath-preserving). This makes `go install go.use-charter.dev/charter/cmd/charter@latest` resolve — the fourth install path alongside brew, binary, and source.
  4. **Reproducibility.** `infra/package.json` pins `wrangler` (4.100.0) + `@cloudflare/workers-types`; `bun run typecheck` gates both workers; `deploy:*` scripts wrap `wrangler deploy --config …`. `infra/README.md` is the operator quickstart; `docs/product/DEPLOY.md` remains the full runbook (Mintlify DNS, Pages settings, verification curls).
- Consequences: New root `infra/` (two workers + `package.json`/`tsconfig.json`/`README.md`) and `web/wrangler.toml` — the first committed Cloudflare config in the repo; `infra/node_modules` is gitignored. The edge topology documented in `DEPLOY.md` becomes executable code, and its stale Step-5 route table (`use-charter.dev/docs*` + `/rules*` only) is corrected to the `use-charter.dev/*` catch-all the Landing section already assumes. Carry-forward **CF-4 is closed in-repo**: the meta-serving worker exists and type-checks; full resolution still requires the live deploy + the apex/`go.` DNS, which is now a `wrangler deploy` away. No Charter **core** change — this is web/edge only; offline-first (ADR-0001), no-LLM-in-core, and the single Go module (ADR-0002) are untouched. The vanity path stays decoupled from the VCS host (ADR-0016), so a future repo move costs no import-path change. Out of scope / deferred: Workers Static Assets migration (recorded as the future direction), `www.` handling and apex redirect policy, and any paid hosted tier. Related: ADR-0001 (offline-first core), ADR-0002 (single Go module), ADR-0006 (latest-docs-first — sources verified 2026-06-12), ADR-0016 (release pipeline & supply chain — deferred the `go install` vanity redirect this closes), ADR-0025 (report — adjacent web design system). This is Slice 19; it depends on Slice 18 (Mintlify docs) being live.
