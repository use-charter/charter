# ADR-0012 Agent Config Scanning

- Status: Accepted
- Context: Charter's two remaining core rules are `AE-CC-001` (dangerous shell commands in agent hook configs — Blocker, OWASP MCP Top 10 beta MCP05 Command Injection & Execution) and `AE-CC-002` (agent edit scope must be explicitly constrained — High, OWASP MCP02 Privilege Escalation via Scope Creep; the architecture doc's older "Permissioning Failures" label). Agent hook configuration exists across several formats (verified live 2026-05-31): Claude Code uses JSON `hooks` blocks in `.claude/settings.json` / `.claude/settings.local.json` (with nested `{type:"command", command, args}` handlers); Cursor uses `.cursor/hooks.json` (`version:1`, `hooks.<event>[].command`); and Git-hook managers use Pkl (`hk.pkl`), YAML (`.pre-commit-config.yaml`, `lefthook.yml`), or shell dirs (`.husky/`). Two design tensions must be resolved: (1) dangerous-command detection is false-positive-prone for shell operator chaining (`&&`, `;`, `$(…)`), and (2) AE-CTX-001 already requires an "edit boundaries" content signal, so AE-CC-002 needs a stricter, non-duplicative contract.
- Decision: (1) `AE-CC-001` scans **JSON hook config files only** in v1 — `.claude/settings.json`, `.claude/settings.local.json`, `.cursor/hooks.json` — via the standard-library `encoding/json`. It walks the parsed `hooks` structure and collects every `command` string and `args` entry, then flags a **high-confidence** dangerous-command set: destructive operations (`rm -rf`, `git reset --hard`, `git clean -fd`, `dd `, `mkfs`, `truncate`) and privilege escalation (`sudo `, `chmod 777`, `chown -R `). Operator-chaining and command-substitution injection detection (`&&`, `||`, `;`, `$(…)`, backticks) is **deferred** because it is false-positive-prone and needs context-aware analysis; this mirrors the high-confidence, low-false-positive posture of `AE-SEC-001`. Pkl/YAML/shell hook managers (`hk.pkl`, `.pre-commit-config.yaml`, `lefthook.yml`, `.husky/`) are deferred. (2) `AE-CC-002` checks the agent context (the `agentcontext` files) for an **explicit, concrete off-limits-path declaration** — recognized sensitive-path tokens declared off-limits (`.env`, `secrets`, `.github/workflows`, `terraform`, `db/migrations`, `credentials`, or `infra`) or a reference to `PERMISSIONS.md`. It flags High when no concrete off-limits-path declaration is found. This is intentionally stricter than `AE-CTX-001`'s edit-boundary signal, which accepts any edit-scope mention. (3) Severity/scoring: `AE-CC-001` is Blocker and engages the existing blocker score cap (≤ 59) with no special per-finding cap; `AE-CC-002` is High; both carry structured `path:line` (or file-level) locations per ADR-0009. (4) The `--format markdown` renderer is additive output per the architecture doc's T1.1.4 — a GitHub-PR-comment-friendly projection of the same result, with no scoring or behavior change.
- Consequences: a new `internal/rules/agentconfig` package holds the two checks and the JSON hook-config discovery/parse (reusing the MCP scan pattern and the shared `agentcontext` registry); a new `internal/render/markdown` package plus a `--format markdown` flag value land in the CLI; the `AE-CC-001`/`AE-CC-002` specs move from stub to as-built contracts; with these two rules the **12 core v1 rules are complete**. Deferred work is recorded for later slices: operator-chaining / command-substitution injection detection, and YAML/Pkl/shell hook-manager scanning. Malformed JSON hook configs fail fast with a wrapped error, consistent with the offline-first, fail-fast posture and the MCP scanner. Related: ADR-0001 (offline-first core), ADR-0004 (contract-first interfaces), ADR-0006 (latest-docs-first), ADR-0009 (finding location), ADR-0011 (MCP config scanning).
