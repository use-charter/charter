# ADR-0020 `charter fix` Engine, Fixer Set & AE-CI-002 Generalization

- Status: Accepted
- Context: `charter fix` is the diff-first repair command (architecture §1.8, M1.4 T1.4.2): "`charter fix --dry-run` shows a unified diff of all auto-fixable findings before applying; `charter fix` applies only safe, reversible changes; backups land in `.charter/backups/` before every write; it never deletes files, never silently mutates, and never auto-fixes secret rules (Commitment #4/#9, ADR-0005)." T1.4.2 scopes the v1 fixers to "AGENTS.md, .gitignore, GitHub Action workflow only — complex rewrites deferred." `charter init` (Slice 11, ADR-0019) deliberately deferred *existing-file* edits (e.g. appending to a present `.gitignore`) to `fix`. Grounded against the code: `doctor.Run` returns `Result.Findings []findings.Finding` (`RuleID`, `Severity`, `Locations`, …); `internal/scaffold` already builds the `AGENTS.md` and `.gitignore` templates. A dogfooding discovery while scoping the GitHub-Action fixer: **`AE-CI-002`'s coverage detection is hardcoded to Charter's own `moon` commands** — `markCoverage` only credits `moon run :check`/`:actionlint`/`:zizmor`/`:security`, so a non-`moon` user repo with excellent CI (running `go test`/`actionlint`/`zizmor`/`govulncheck` directly) can never satisfy repo-quality/workflow-lint/security coverage and AE-CI-002 false-positives on essentially every external repo — the same over-fitting class as the `AE-CTX-002` `moon run :check` FP (Slice 11) and the SLSA `AE-CI-002` FP (Slice 9).
- Decision:
  1. **`internal/fix` engine.** A pure planner + a thin applier. `Plan(result doctor.Result, root string, opts) []FilePlan` maps each fixable finding (by `RuleID`) through a registry to a `FilePlan{Path, Action(create|append), NewContents, Diff}`, where `Diff` is a hand-rolled **unified diff** (v1 fixes are create or append, so diffs are all-`+` hunks — no diff dependency). The applier writes plans: it **backs up any existing target** to `.charter/backups/<RFC3339-ish-timestamp>/<path>` before writing, then writes `0o644`; it **never deletes/truncates an unrelated file** and **never targets a secret rule** (AE-SEC-001/002, AE-CC-001 are not fixable — secrets and dangerous-command remediation are manual by Commitment #9). `--dry-run` prints the diffs and writes nothing.
  2. **Fixer set (v1):** `AE-CTX-001` → create `AGENTS.md` (reuse `scaffold.AGENTSMarkdown`; create-only — a present-but-weak file is a deferred complex rewrite); `AE-CTX-004` → if `.gitignore` is absent create it (`scaffold.Gitignore`), else **append only the missing agent-artifact patterns** to the existing file (the diff-first existing-file edit `init` deferred); `AE-CI-002` → create `.github/workflows/charter.yaml` adding the Charter CI gate. Each fixer is pure (returns contents/plan); the engine owns I/O. Deferred: `AE-ENV-001` (its missing piece is usually an *opinionated* hook config, not a file create), `AE-MCP-001` (pin target needs Slice 13's catalog), and all secret/dangerous rules (never auto-fixed).
  3. **Generalize `AE-CI-002` coverage detection** (fixes the cross-vendor FP and lets the scaffolded workflow + a user's existing CI clear the rule). `markCoverage` recognizes, in addition to the `moon run :*` forms: repo-quality = a test/build command (`go test`/`go build`, `npm test`/`npm run test`, `pnpm`/`yarn` test, `cargo test`/`cargo build`, `pytest`/`python -m pytest`, `bun test`, `make test`/`make check`); workflow-lint = `actionlint` **and** `zizmor` each present as a direct command **or** a recognizable action `uses:` (`rhysd/actionlint`, `zizmorcore/zizmor`); security = a recognized scanner (`govulncheck`, `osv-scanner`, `gitleaks`, `trivy`, `grype`) as a command or action (`github/codeql-action`). The charter-product-gate (`charter doctor` / `use-charter/charter-action@`) is unchanged — it remains AE-CI-002's anchor. Charter's own moon-based CI keeps passing.
  4. **Exempt the first-party action from the unpinned-action check.** `unpinnedActionEvidence` no longer flags `use-charter/charter-action@<tag>` (tag-pinned is the conventional consumer form for a first-party action), alongside the existing `slsa-github-generator` exemption (ADR-0016). This lets the scaffolded `charter.yaml` use `use-charter/charter-action@v1` without tripping AE-CI-002.
  5. **The `AE-CI-002` fixer scaffolds a focused workflow**, not an all-in-one: `charter.yaml` checks out (SHA-pinned `actions/checkout`), runs `use-charter/charter-action@v1` (which runs `charter doctor` and uploads SARIF), with least-privilege `permissions` (`contents: read`, `security-events: write`). It supplies the **charter-gate**; a repo's *existing* test/lint/security workflows now count (via the generalization), so AE-CI-002 fully clears when they exist and correctly leaves genuine gaps otherwise (the fixer never fabricates a user's test/lint/security CI — mirroring `init`'s create-missing discipline).
  6. **CLI:** `charter fix [--rule AE-XXX-NNN] [--dry-run] [--all] [--yes] [--path]`. Default lists fixable findings + their diffs and applies them after writing backups; `--dry-run` previews; `--rule` targets one; exit 0 on success, 2 on error. Output names each created/appended file and the backup location.
  7. **Armed, not fired:** the scaffolded `charter.yaml` references `use-charter/charter-action@v1`, which is published at launch (Slice 17) — the workflow is correct for post-launch consumers and statically clears the (generalized) AE-CI-002 now.
- Consequences: new pure `internal/fix` package (registry + plan + diff) and `cmd/charter/fix.go` (+ `root.go`); `internal/rules/ci/ci002.go` gains generalized coverage matchers + the first-party-action exemption, with the AE-CI-002 spec + audit checklist updated; `.charter/backups/` is written on apply (already git-ignored by `init`'s `.gitignore` block / Charter's own). Reuses `internal/scaffold` for the AGENTS.md/.gitignore contents. An integration test proves a repo with normal CI but no Charter clears AE-CI-002 after `fix`, and that `fix AE-CTX-004` appends patterns with a backup + diff (measured: `fix` raises a representative non-moon repo from 91 to 96, clearing AE-CTX-004 + AE-CI-002). No network, no LLM, no new dependency. Deferred: `AE-ENV-001`/`AE-MCP-001` fixers, present-but-weak-file rewrites, content-aware (3-way) diffs, interactive selection. Related: ADR-0005 (diff-first), ADR-0019 (init/scaffold), ADR-0016 (SLSA exemption precedent), ADR-0008 (score); architecture §1.8 + M1.4 T1.4.2. The AE-CI-002 generalization continues the dogfooding-driven FP-reduction line (Commitment #9) begun in Slices 9 and 11.
- Refinement (2026-06-02, CF-6): the engine gained a third action, **`Replace`** (a surgical, content-preserving in-place rewrite: back up the existing file, then write the new bytes), and a **multi-plan fixer** path (`multiRegistry`) for rules that touch several files. The first consumer is an **`AE-MCP-001` fixer** that bumps an MCP server package to a catalog-known *safe* version — an advisory-affected pin to its `fixedIn`, or an unpinned/behind-stable cataloged package to the catalog `stableVersion` — by replacing only the exact quoted package-spec token (`internal/rules/mcp.PlanPins` derives the safe rewrites from the catalog; ADR-0021). It **never** rewrites a deprecated/archived package (the successor is a different package/transport, so migration stays manual) and never touches anything but the version token. This is the bounded, safe slice of the version-pin fixer; the broader content-aware rewrites (present-but-weak files, 3-way diffs) remain deferred. `AE-ENV-001` stays unfixed (CF-5, opinionated hook config).
