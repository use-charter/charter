# ADR-0016 Release Pipeline & Supply-Chain Provenance

- Status: Accepted
- Context: Charter's architecture (§1.8, M1.5 T1.5.2) commits to dogfooding the supply-chain posture it scores in others (Commitment #6): "GoReleaser v2 producing SLSA L2 provenance, cosign keyless signatures, and SPDX SBOMs for all release artifacts." Today `.github/workflows/release.yml` is a build-only stub (`moon run :build`, `permissions: contents: read`) and nothing is signed, attested, checksummed, or distributed. The GitHub Action (Slice 10) needs a real, versioned, verifiable binary to download, and `charter version` (ADR-0017) needs build metadata injected at release time. Two repo realities shape this: (1) the module path is the **vanity** `go.use-charter.dev/charter` while the live remote is `github.com/tashfiqul-islam/charter` — so the GitHub release/identity owner and the eventual `use-charter` org may diverge; (2) the repo pins **all** tooling in `mise.toml` (aqua backend) and drives work through Moon tasks, and CI is zizmor `--pedantic` clean (every third-party action SHA-pinned). Verified live (2026-06-01) against official sources: **GoReleaser v2.16.0** (the `brews` block is deprecated since v2.10 in favor of `homebrew_casks`; v2 renamed `snapshot.name_template` → `version_template`); **cosign v3.0.6** (v3 standardized on a single `--bundle <artifact>.sigstore.json` for `sign-blob`/`verify-blob`; `id-token: write` lets cosign auto-detect the GitHub OIDC token, no keys); **slsa-github-generator v2.1.0** `generator_generic_slsa3.yml`, which consumes GoReleaser's base64 artifact hashes and produces **SLSA Build L3** provenance from an isolated reusable workflow (exceeds the doc's L2 target); SBOMs via **syft** (invoked by GoReleaser `sboms`).
- Decision:
  1. **GoReleaser v2 as the release engine, pinned in mise and driven through Moon.** Add `goreleaser`, `cosign`, and `syft` to `mise.toml` (aqua) and a root `.goreleaser.yaml` (`version: 2`). New Moon tasks: `release` (`goreleaser release --clean`, the tag-triggered path), `release-snapshot` (`goreleaser release --snapshot --clean --skip=sign,publish`, the offline local/CI dry-run), and `release-check` (`goreleaser check`, config validation). `release-check` joins the `:check` dependency set so a broken release config fails the normal gate; `release`/`release-snapshot` stay out of `:check` (they build artifacts / need network). This keeps local↔CI parity and dogfoods toolchain pinning — no `goreleaser-action`/`cosign-installer` actions needed (mise provides the binaries).
  2. **Builds:** `./cmd/charter` → `charter`, `CGO_ENABLED=0`, `-trimpath`, `mod_timestamp={{.CommitTimestamp}}` (reproducible); `linux/darwin/windows` × `amd64/arm64`. ldflags inject the build stamp into `internal/version` (ADR-0017): `-s -w -X …/internal/version.injected={{.Version}} -X …/internal/version.commit={{.FullCommit}} -X …/internal/version.date={{.CommitDate}}`.
  3. **Checksums + SBOM:** a single `checksums.txt` (sha256 — required by the SLSA generic generator); `sboms` via syft emitting SPDX-JSON per artifact. We emit the **highest SPDX version syft produces cleanly** and **reconcile the architecture doc to the as-built version** rather than over-claim "SPDX 3.0.1" — SBOM accuracy over a version label.
  4. **Keyless signing (cosign v3 bundle):** GoReleaser `signs` runs `cosign sign-blob --bundle ${artifact}.sigstore.json --yes ${artifact}` over `checksums.txt` (so one signature covers every artifact by hash). No keys, no secrets — Fulcio/Rekor via the workflow's GitHub OIDC identity. Verification: `cosign verify-blob --bundle … --certificate-identity <release-workflow-ref> --certificate-oidc-issuer https://token.actions.githubusercontent.com`.
  5. **SLSA L3 provenance:** a second workflow job calls `slsa-framework/slsa-github-generator/.github/workflows/generator_generic_slsa3.yml@v2.1.0` with `base64-subjects` = GoReleaser's artifact hashes, `upload-assets: true`. This reusable workflow **must be referenced by its semantic tag** (slsa-verifier resolves the builder identity from the tag), which is the **single, documented exception** to the repo's SHA-pin rule — recorded with an inline zizmor ignore (`unpinned-uses`) carrying this ADR's number as justification. All other actions stay full-SHA-pinned.
  6. **Distribution = signed GitHub Release + Homebrew cask.** GoReleaser `homebrew_casks` (not the deprecated `brews`) publishes to a `homebrew-tap` repo. Because Charter binaries are cosign-signed but **not Apple-notarized**, the cask carries a post-install hook that strips the `com.apple.quarantine` attribute (notarization deferred — it needs an Apple Developer cert). `go install` via the vanity path is **out of scope here**: it requires `go.use-charter.dev` to serve a `go-import` meta tag, tracked with the web/launch slices.
  7. **Parametric identity (no hardcoded owner).** The workflow and `.goreleaser.yaml` resolve owner/repo from `${{ github.repository }}` / `${{ github.repository_owner }}` and a configurable tap (`TAP_OWNER`/`HOMEBREW_TAP_TOKEN` env), so neither `use-charter` nor `tashfiqul-islam` is baked in — a later org move changes the cosign cert identity string but not the pipeline design.
  8. **Armed, not fired.** This slice delivers and validates everything verifiable **without** writing to the public Rekor log or cutting a real release: `goreleaser check`, an offline `--snapshot` build of all targets, `charter version`, actionlint/zizmor on the new workflow, and `moon run :check`. The first **real** signed release (a pre-release `v0.9.0`, `release.prerelease: auto`) — exercising live cosign/Fulcio/Rekor, SLSA provenance, and the tap push — is a go-public ops action gated on the org/tap/secrets existing. No tag is pushed in this slice.
- Consequences: `release.yml` is rewritten into two least-privilege jobs — `goreleaser` (`contents: write` + `id-token: write`) and `provenance` (`actions: read`, `id-token: write`, `contents: write`); top-level `permissions: contents: read` retained. New root `.goreleaser.yaml`; `mise.toml`/`mise.lock` gain `goreleaser`/`cosign`/`syft`; root `moon.yml` gains `release`/`release-snapshot`/`release-check` (+ `:check` dep). A `homebrew-tap` repo and a `HOMEBREW_TAP_TOKEN` secret are **out-of-band founder/admin prerequisites** for the real publish; snapshot builds skip the tap. The repo can now produce signed, attested, SBOM-bearing releases that satisfy AE-CI-002/AE-ENV-001 it enforces on others, and Slice 10's Action gets a verifiable binary. Deferred: live tag/`v0.9.0` cut, Apple notarization, `go install` vanity redirect, GHCR/container images, Scoop/Nix distribution. Related: ADR-0001 (offline-first core), ADR-0002 (single Go module), ADR-0006 (latest-docs-first), ADR-0017 (version command & build stamping); architecture §1.8 + M1.5 T1.5.2.
