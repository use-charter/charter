# ADR-0010 Trademark Clearance

- Status: Accepted — **PROCEED as "Charter"** (informed-risk acceptance, 2026-06-13; supersedes the prior "Pending external clearance")
- Context: The architecture made name clearance a hard gate before public launch (this ADR was the blocker). A preliminary knock-out search was run (2026-06-13). **Federal (USPTO):** no obvious exact-match live `CHARTER` registration for CLI / CI / developer tools in Class 9/42; adjacent software/IT marks exist but are telecom/ISP/support (Charter/Spectrum) or compound (`ASPEN POWERED BY CHARTER`, Charter Software) — medium-low. **Common-law:** a direct same-niche collision was found and **verified against live sources**, not just an LLM summary: the GitHub org `Stackbilt-dev` ships `Stackbilt-dev/charter` — *"Local-first AI agent governance CLI. Modular `.ai/` context system (ADF) replaces monolithic CLAUDE.md files"* — and npm `@stackbilt/cli` **v1.0.0** (*"Charter CLI — repo-level governance toolkit"*) + `@stackbilt/core` v1.0.0. That is effectively Charter's product (local-first AI-agent-governance CLI, repo/CLAUDE.md context, a `charter` command, same developer audience), already published at v1.0. Two facts informed the call: (1) **Apache-2.0 ≠ trademark protection** — the OSS license governs copyright, not name confusion; (2) **`use-charter.dev` differentiates the domain/handle, not the product name** — the tool is still "Charter", invoked as `charter`, which is the shared element with Stackbilt.
- Decision: **Ship as "Charter" under the `use-charter` org and `use-charter.dev` domain**, accepting the residual risk with eyes open, rather than rename. Rationale: "Charter" is a common, weak/descriptive term (narrow trademark scope); both parties are small free OSS; the two tools live in **different distribution ecosystems** (Charter = Go binary via Homebrew/`go install`; Stackbilt = npm), so the `charter` command clash is an edge case (only if a user installs both); founder is already invested in the domain + brand; and a free OSS tool can rename later if forced. The differentiators we lean on: the `use-charter` org/handle and `use-charter.dev` everywhere, distinct positioning ("AI-agent readiness, scored" vs. governance toolkit), and the path-namespaced module `go.use-charter.dev/charter`.
- Consequences: The name gate is **closed** — public launch, the `use-charter.dev` go-live (ADR-0026), org domain verification (CF-10), and brand lock-in may proceed as "Charter". **Accepted residual risk, recorded for due diligence:** (a) market/SEO confusion with a published same-niche "Charter"; (b) a possible future cease-and-desist from Stackbilt or a larger "Charter" holder; (c) a blocked/contested federal registration if we ever file. **Rename trigger (contingency):** if Stackbilt (or another holder) formally asserts, if we pursue a federal trademark and it's refused on likelihood-of-confusion, or if Charter goes commercial/funded — revisit and execute a rename then (binary name, module path, domain, org, brand; the `AE-*` rule IDs are generic and survive). Not pursuing a paid attorney clearance opinion at this stage (free OSS, pre-revenue); revisit if commercializing. Before any federal filing, run the official USPTO queries in the new Trademark Search tool (`CM:charter AND LD:true AND IC:009` / `IC:042`). Related: ADR-0026 (go-public deploy under use-charter.dev), carry-forward CF-10 (org domain verify).
- Exit criteria: **met** — this ADR now records the resolved outcome (PROCEED). Re-open only on a rename trigger above.
