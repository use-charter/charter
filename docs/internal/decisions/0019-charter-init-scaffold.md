# ADR-0019 `charter init` Scaffold Model

- Status: Accepted
- Context: `charter init` is the onboarding command (architecture Â§1.8, M1.4 T1.4.1): "scaffold all required agent context files from a blank repo in under 2 minutes â€¦ `charter init` then `charter doctor` scores â‰Ą 80 out of the box for Go projects." It is the antidote to the "one-time setup trap" the architecture's validation-failure analysis names. It must be deterministic, offline, and honor the Ten Commitments â€” especially #3 (never delete a user file) and #4 (never silently mutate; diff-first). The rule code dictates exactly what a passing scaffold must contain: `AE-CTX-001` (Blocker) needs an agent context file with project-summary + tech-stack + edit-boundary + verification-command signals, â‰Ą 5 non-empty lines, and â‰¤ 600 estimated tokens; `AE-CC-002` (High) needs concrete off-limits tokens (`.env`, `secrets`, `.github/workflows`, â€¦); `AE-CTX-004` (Medium) needs `.gitignore` agent-artifact patterns; `AE-CTX-002` (Medium) checks `AGENTS.md` against repo truth. Verified live (2026-06-01): **`AGENTS.md`** is the Linux-Foundation/agents.md community standard â€” freeform Markdown, the *universal* file that Cursor, Copilot, Claude Code, etc. all read (so per-tool context files are redundant); common sections are Project Overview, Tech Stack (versioned), Commands, Boundaries. **`.claude/settings.json`** uses `{$schema: https://json.schemastore.org/claude-code-settings.json, permissions:{allow/ask/deny}}` with rules like `Read(./.env)` / `Read(./secrets/**)` in `deny`; MCP is configured separately (none needed). A dogfooding catch: **`AE-CTX-002` hardcodes `moon run :check`** as a required `AGENTS.md` marker â€” Charter-repo-specific, contradicting its own spec ("stated verify command vs current CI config"), so it false-positives on every non-`moon` repo (including ones `init` generates).
- Decision:
  1. **Create-missing-only; never overwrite or delete.** `init` writes a file only when it is absent; an existing file is **skipped and reported**, never modified or removed (Commitments #3/#4). Surgical edits to *existing* files (e.g. appending agent patterns to a pre-existing `.gitignore`) are `charter fix`'s job (Slice 12, diff-first). On a genuinely blank repo this yields a clean scan; on a partially-set-up repo `init` fills the gaps and leaves the rest to `fix`.
  2. **`--dry-run` previews** the file plan (create/skip per file) and writes nothing â€” Charter's diff-first idiom applied to scaffolding. `init` is otherwise **non-interactive and deterministic** (no prompts); `--yes` is accepted as the implicit default for scripted/CI use.
  3. **Scaffold set:** `AGENTS.md` (agents.md-standard sections, tuned to pass `AE-CTX-001`/`AE-CC-002`: Project Overview, Tech Stack, Commands with `charter doctor` verification, Edit Boundaries listing `.github/workflows/`/`.env*`/`secrets/`/`db/migrations/`/`terraform/`; â‰¤ 600 tokens), `charter.yaml` (`policy.profile` from `--profile`, default `standard`; schema-valid per `schemas/charter-config.schema.json`), `.gitignore` (agent-artifact patterns for `AE-CTX-004` â€” created only if absent), `ARCHITECTURE.md` (overview template), `.env.example` (placeholder), and `.claude/settings.json` (MCP-free, with `Read(./.env)`/`Read(./.env.*)`/`Read(./secrets/**)` denies) when Claude is detected or requested. **`AGENTS.md` is the universal context** (Cursor/Copilot read it natively) â€” no redundant per-tool context files are generated.
  4. **Deterministic, offline detection** from tracked manifests: language (robust Go via `go.mod` + its `go`/`toolchain` version; best-effort JS via `package.json`, Python via `pyproject.toml`, Rust via `Cargo.toml`), CI (`.github/workflows`), agents (`.claude`/`.cursor`/`.github/copilot-instructions.md`). Detection populates an accurate `AGENTS.md` so `AE-CTX-002` (repo-truth) holds. No network, no LLM.
  5. **Flags:** `--path` (target; default CWD), `--profile` (`strict|standard|relaxed`, default `standard`), `--agents` (csv; default = detected, falling back to `claude`), `--dry-run`, `--yes`. Exit 0 on success (incl. all-skipped); exit 2 on error.
  6. **Fix `AE-CTX-002`'s verification marker** (a directly-related, dogfooding-surfaced FP): extract the verification-command signal set into a shared helper and have `AE-CTX-002` accept exactly what `AE-CTX-001` already accepts (`moon run :check`, `charter doctor`, "verification command", "verify with") instead of hardcoding `moon run :check`. This aligns the two CTX rules, matches `AE-CTX-002`'s own spec, and stops the false positive on every non-`moon` repo. The generic off-limits markers (`.env*`, `secrets/`) and the conditional markers (`hk.pkl`/architecture-doc only when present) are unchanged.
- Consequences: a new pure `internal/scaffold` package (detection + template builders returning file contents + a create/skip file plan; unit-tested, deterministic, â‰¤ 600-token `AGENTS.md` asserted) and `cmd/charter/init.go` (+ `root.go` registration); `internal/rules/context` gains a shared verification-signal helper used by `AE-CTX-001` and `AE-CTX-002`, with the `AE-CTX-002` spec + audit checklist note updated. An `init`â†’`doctor` integration test on a synthesized blank Go repo asserts the OOTB score (â‰Ą 80; **measured 95** as built, after the CTX-002 alignment + a `.gitignore` whose agent-artifact block includes `.hk/` and `.env*` plus `!.env.example` to fully clear `AE-CTX-004` â€” residual `AE-ENV-001`/`AE-CI-002` only). No network, no LLM, no new dependency. Deferred to `charter fix` (Slice 12): modifying *existing* files (e.g. `.gitignore` append, `mise.toml` creation for `AE-ENV-001`, MCP pinning) with diffs; deferred otherwise: `.cursor/rules` scaffolding (AGENTS.md is the universal source), interactive prompts, and `.env.example` codebase-scanning (a minimal placeholder ships).
