# ADR-0024 Interactive TUI & Terminal-Output Architecture

- Status: Accepted
- Context: Charter's output today is plain `fmt.Fprintf` text plus the `json`/`markdown`/`sarif` renderers ‚Äî correct and pipe-safe, but visually flat, and there is no interactive surface and no terminal-capability discipline (no `NO_COLOR`/TTY detection, no color tiers, no hyperlinks). The canonical architecture's ¬ß1.8 command gallery already *depicts* pristine styled output (the `‚ú¶ charter` mark, per-category scan bars, boxed finding cards, a score hero) ‚Äî that aspiration is unimplemented. 2026 CLI standards make two requirements non-negotiable and mutually constraining: human-first styled output **in a TTY**, and machine-clean, **non-interactive** structured output when piped or driven by an agent/CI. Grounding (fetched 2026-06-02, ADR-0006): [clig.dev](https://clig.dev/) (human-first, stdout/stderr split, fast startup, respect the terminal); the CLI Spec ([clispec.dev](https://clispec.dev/)) and WebCLI ‚Äî *structured output when piped, human output in a TTY, non-interactive by default, never block without a TTY*; [no-color.org](https://no-color.org/) ‚Äî when `NO_COLOR` is set to any non-empty value an app MUST NOT add color (honor absolutely); [terminfo.dev](https://terminfo.dev/fundamentals/color-detection) ‚Äî color precedence and degradation (truecolor‚Üí256‚Üí16‚Üímono via `COLORTERM`/`TERM`/`isatty`) and OSC 8 hyperlinks (supported by Ghostty/iTerm2/WezTerm/foot/VTE/Windows Terminal; **not** Apple Terminal.app ‚Äî degrade to plain text). The Go TUI standard in 2026 is the Charm stack, now **stable** on the `charm.land` module paths (verified via the module proxy 2026-06-02): `charm.land/bubbletea/v2 v2.0.7` (Elm/MVU; "Mode 2026" synchronized output by default), `charm.land/lipgloss/v2 v2.0.3` (pure styling ‚Äî Bubble Tea owns I/O), `charm.land/bubbles/v2 v2.1.0` (components), `charm.land/fang v1.0.0` (Cobra shell: styled help/usage/errors, `--version`, man-page generation). Visual direction is fixed by the design references committed at `docs/internal/designs/charter-doctor-init-fix.html` and `docs/internal/designs/charter-supress-report-version.html` (terminal-window chrome, `‚ùØ` prompt, braille spinner, `‚ñà‚ñà‚ñà‚ñà‚ñà‚ñà`/`¬∑¬∑¬∑¬∑¬∑¬∑¬∑¬∑¬∑¬∑` scan bars, severity badge pills, finding cards, score hero, `init` stepper, `fix` diff + apply prompt, `suppress` tables, `version` supply-chain block, per-screen key-hint footer; Tabler-style glyphs with ASCII fallback), themselves informed by 2025‚Äì2026 agent CLIs (opencode, Claude Code, Codex, Gemini CLI). Hard constraint: Charter is deliberately near-zero-dependency (3 direct deps today) and deterministic/offline (Commitments #4/#7), so any TUI must be **contained** and must **never** alter the non-interactive contract that agents and CI depend on.
- Decision: add a styled-and-interactive terminal presentation layer, strictly TTY-gated, on top of the unchanged non-interactive contract. All detection stays offline/deterministic (no network, no LLM).
  1. **Adopt the `charm.land` v2 stack** ‚Äî `bubbletea/v2` + `lipgloss/v2` + `bubbles/v2` ‚Äî plus **`fang`** wrapping the Cobra root for styled help/usage/error output, `--version`, and man pages. `glamour` is added **only if** markdown is rendered inside the TUI. Charm is confined to the interactive/TTY presentation path. The `--format {json,sarif,markdown}` renderers and piped/`--quiet` text remain **pure stdlib** ‚Äî no Charm, no ANSI ‚Äî so CI/agent/pipe output is byte-for-byte unchanged.
  2. **`charter doctor` stays non-interactive by default** (clig.dev / CLI Spec: non-interactive by default; never block without a TTY). It emits **styled** text when stdout is a TTY and **plain** text when piped or when `NO_COLOR`/`--no-color`/`TERM=dumb`/CI is detected. The visual language matches `docs/internal/designs/*.html`.
  3. **Interactive TUI is opt-in via `charter doctor -i/--interactive`** ‚Äî it errors clearly (exit 2) if stdout is not a TTY. It is a master-detail browser over a **single** scan result: a findings list (groupable/sortable by severity & category) + a detail pane (`RuleID`, severity, category, summary, evidence, remediation, `path:line`, and the `charter explain` rationale), a header carrying the score + category scorecard, and a `bubbles`-driven key-hint footer. Interactions: filter by severity/category, `/` text search, vim + arrow navigation, `y` to copy `path:line` (OSC 52), `r` to re-run the scan in place, `?` help, `q` quit. **No** always-on file-watch and **no** `$EDITOR` spawn in v1 (deferred ‚Äî Phase 1.5).
  4. **Terminal-output discipline** (the default styled path). Color precedence, highest first: explicit `--color=always|never|auto` / `--no-color`; then `NO_COLOR` (any value ‚Üí off); then `TERM=dumb` ‚Üí off; then `!isatty(stdout)` ‚Üí off; then `COLORTERM=truecolor|24bit` ‚Üí truecolor; then `*-256color`/`xterm|screen|tmux` tiers; degrade truecolor‚Üí256‚Üí16‚Üí**mono** (in mono, hierarchy via bold/inverse/underline). OSC 8 hyperlinks (rule id ‚Üí `helpUri`, `path:line` ‚Üí `file://`) are emitted **only** when color/links are enabled, and never when piped. Severity is always carried by a **text label**, never color alone (accessibility); glyphs have ASCII fallback.
  5. **`charter explain <RULE>`** prints a rule's catalog entry (id, name, category, severity, rationale, remediation, `helpUri`, grounded references) ‚Äî `text` default, `--format json` optional. It is the single rule-explanation surface reused by the TUI detail pane (ADR-0024) and the HTML report (ADR-0025), sourced from `internal/rules/catalog` (ADR-0014). **`charter doctor --rule <ID>[,<ID>‚Ä¶]`** filters the scan to named rules (already shown in the ¬ß1.8 gallery).
- Consequences: new internal packages ‚Äî `internal/terminal` (capability detection: `isatty`, `NO_COLOR`, `COLORTERM`/`TERM` tier, OSC 8 capability, the Lipgloss adaptive palette mapped from the design tokens `--color-text-{primary,secondary,tertiary}` / `{success,danger,warning,info}` and their background/border variants), `internal/render/text` (the styled non-interactive renderer + its mono/plain fallback), `internal/tui` (the Bubble Tea program/models for `doctor -i`), and `internal/explain` (over `internal/rules/catalog`). `cmd/charter` gains `-i/--interactive`, `--color`, `--no-color`, and `--rule` on `doctor`, a new `explain` command, and `fang` wrapping the root. `go.mod` gains the `charm.land` v2 modules (plus transitive `termenv`/`x-ansi`/`x-term`) ‚Äî Charter's first significant dependency expansion; justified by UX, contained to the TTY path, and **guarded by a contract test** asserting zero ANSI and zero Charm involvement on non-TTY / `--format {json,sarif,markdown}` / `NO_COLOR` paths (the byte-stable agent/CI contract). Determinism: the TUI is excluded from golden tests; the styled text path is snapshot-tested per color tier; the `json`/`sarif` goldens are untouched and the score formula (ADR-0008) is unchanged. Commitments preserved: no LLM, offline (#4); no network (#7); non-interactive default for agents/CI. Architecture ¬ß1.8 + the command-surface text are amended: the interactive TUI surface is added, and `charter explain`, `--no-color`/`--color`, `doctor --rule`, and `report --format html` move from the "Phase 1.5 / v1.1 preview" addendum into the v1.0 surface. Visual source of truth: `docs/internal/designs/*.html`, with colors/fonts/skills canonical in `docs/internal/designs/DESIGN-TOKENS.md`. **Fonts are terminal-inherited** ‚Äî the TUI sets no typeface (color/layout/glyphs only); the font system governs the HTML report + web/docs, not the terminal. Palette tiers/contrast are grounded with the installed `ui-ux-pro-max` + `design-taste-frontend` + `visual-design-foundations` skills, with `interaction-design` for state/feedback (full map in `DESIGN-TOKENS.md`). Related: ADR-0006 (latest-docs-first ‚Äî all versions fetched fresh), ADR-0014 (rule catalog reused by `explain`), ADR-0008 (score formula, unchanged), ADR-0025 (report delivery ‚Äî shares the finding/category/explain model and the design tokens).
