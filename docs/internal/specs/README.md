# Rule Specs

Each rule spec should define:

- Rule ID
- Severity
- Category
- Description
- Detection logic
- Pass example
- Fail example
- Remediation
- Related ADRs
- Related evals

Use `None yet` when traceability targets do not exist yet. Do not omit fields.

Update a rule spec whenever rule semantics, remediation, output shape, or error wording changes.
