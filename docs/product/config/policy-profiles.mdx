---
title: "Policy Profiles"
description: "Use policy profiles to set a consistent score threshold across a team or repo without per-run flags."
---

A policy profile is a named preset that sets the minimum passing score for your repo. Configuring a profile in `charter.yaml` means every invocation of `charter doctor` — on any machine, in any CI run — uses the same threshold without anyone needing to pass a flag.

## What a profile does

A profile sets the score threshold below which `charter doctor` exits with a non-zero status. When the score meets or exceeds the threshold, Charter exits 0. When it falls below, Charter exits 1 and prints the failing findings.

Using a profile in `charter.yaml` is equivalent to passing `--threshold <value>` on every run, but committed to the repo so the threshold is part of the codebase rather than a per-engineer convention.

## Available profiles

| Profile | Threshold | When to use |
|---|---|---|
| `standard` | 80 | Default. A solid baseline for most repos, including active development projects that are progressively adopting agent tooling. |
| `strict` | 90 | A higher bar suitable for security-sensitive repos, production services, or teams that have already resolved the common findings and want to hold the line. |

## Setting a profile in charter.yaml

```yaml
policy:
  profile: strict
```

Place `charter.yaml` in the root of your repo. Charter discovers it automatically.

## Setting an exact threshold

If neither profile maps to the threshold you want, set an explicit value:

```yaml
policy:
  threshold: 85
```

`policy.threshold` overrides the default for the selected profile. You can use it alongside `policy.profile` if you want to name the intent clearly:

```yaml
policy:
  profile: standard
  threshold: 85
```

In this case, `threshold: 85` takes effect — the profile label is documentation only.

## Command-line override

Passing `--threshold` at invocation time wins over everything in `charter.yaml`. This is useful for CI jobs that need a different gate than the repo default, or for one-off checks during development:

```bash
# Temporarily lower the threshold during a migration
charter doctor --threshold 70
```

The `charter.yaml` file is not modified by `--threshold`.

## Threshold precedence

When multiple threshold sources are present, Charter applies them in this order (highest priority first):

1. `--threshold` flag on the command line
2. `policy.threshold` in `charter.yaml`
3. The default threshold for the `policy.profile` (80 for `standard`, 90 for `strict`)
4. Built-in default: **80** (applied when `charter.yaml` is absent or has no policy section)
