**Rule ID:** AE-CI-002 · **Severity:** Low · **Category:** CI · **Auto-fixable:** Yes — `charter fix --rule AE-CI-002`

## Why this rule

A local Charter pass only covers one developer's current state. Without a CI gate, a PR can introduce context drift, a new MCP server, or a weak suppression that no one catches until the next manual scan.

## What triggers it

Charter inspects `.github/workflows/` for three things:

1. **No `charter doctor` step.** If no workflow invokes `charter doctor` (or an equivalent Charter entrypoint), Charter cannot enforce the agent-readiness gate in CI — findings accumulate silently until the next manual scan.
2. **Workflow linting findings.** Charter runs `actionlint` against your workflow files and surfaces any issues it reports — incorrect syntax, invalid event triggers, misused contexts, and similar structural problems.
3. **Unpinned third-party actions.** Actions referenced by a mutable tag (e.g. `uses: actions/checkout@v4`) rather than a full commit SHA can be silently updated by a maintainer. Charter flags any non-pinned action. SLSA reusable workflows from `slsa-framework/slsa-github-generator` are exempt because their trusted-builder identity is resolved through the tag, not a SHA.

## Examples

### Failing

A repo where no workflow calls `charter doctor`, or where `uses: some-org/some-action@v2` appears without a SHA pin — flagged Low.

### Passing

All workflow steps using third-party actions are pinned to a full SHA (e.g. `uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683`), and at least one workflow runs `charter doctor`.

## How to fix

Add a workflow step that runs `charter doctor` so the agent-readiness score is enforced on every pull request. Pin all third-party `uses:` references to a full commit SHA. You can apply the pinning fix automatically with `charter fix --rule AE-CI-002`.

## Edge cases

during pre-implementation bootstrap, CI may legitimately omit `charter doctor` itself if the scanner is not built yet. Once `charter doctor` exists, the repo should run a Charter-related CI gate, not just local quality gates.
