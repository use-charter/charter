**Rule ID:** AE-AUTO-001 · **Severity:** Medium · **Category:** Autonomy · **Auto-fixable:** No

## Why this rule

Even with tests present, an agent needs to know how to run them. A discoverable command closes the agent's work loop — run, observe, fix, repeat — without guessing.

## What triggers it

This rule only applies when at least one code language is active in the repo (see AE-TEST-001). Charter checks whether a test or verification command is discoverable from the repo's tracked files.

A command is considered discoverable if any of the following signals exist:

- A `Makefile` with a `test:` or `check:` target
- A `justfile` or `.justfile` with a `test` or `check` recipe
- A `Taskfile.yml` or `Taskfile.yaml` with a `test:` or `check:` task
- A `moon.yml` task named `test` or `check`
- A `mise.toml` or `.mise.toml` with a `[tasks.test]` or `[tasks.check]` section
- A `package.json` with a `test` key (or a `test:*` variant) under `scripts`

For languages with a well-known conventional test command, no task runner is required: Go repos with `go.mod` satisfy the check via `go test`, Rust repos with `Cargo.toml` satisfy it via `cargo test`, and Python repos satisfy it via `pytest` when a pytest config file is present (`pytest.ini`, `tox.ini`, or a `[tool.pytest.ini_options]` section in `pyproject.toml`).

The finding fires when none of these signals exist for an active language.

## Examples

### Failing

A TypeScript project with source files and tests in place, but no `test` script in `package.json` and no task runner configured — flagged Medium. An agent reading the repo has no reliable way to invoke the test suite.

### Passing

A Go module with `go.mod` — `go test ./...` is the conventional command and is always discoverable, no additional config needed. A JavaScript project with `"scripts": { "test": "jest" }` in `package.json` — passes. A repo using `mise.toml` with a `[tasks.test]` section — passes regardless of language.

## How to fix

expose a test command via a task runner (Makefile/justfile/Taskfile/package.json scripts/mise/moon) so an agent can discover and run it.

## Score impact

`Medium` (−4); no hard cap.

## Edge cases

**N/A** when no language is active. **FP guard:** a single-language Go/Rust/Cargo repo is never penalized for lacking a `Makefile` — its toolchain is the contract. Conventional Python detection requires a pytest config (not merely `.py` files), since `python` alone is not a zero-config test command.
