**Rule ID:** AE-CTX-002 · **Severity:** Medium · **Category:** Context · **Auto-fixable:** No

## Why this rule

An agent that reads a context file describing the wrong stack or stale commands will make confident mistakes — wrong build commands, wrong test paths, wrong edit scope.

## What triggers it

Charter reads the context file and compares its claims against the repo's actual state. The finding fires when the stated tech stack doesn't match the manifests Charter can see, when the stated test command doesn't exist in any task runner or script, or when declared off-limits paths don't match the repo structure.

## Examples

### Failing

The context file references stale runtimes, a test command that no longer exists, or edit-scope boundaries that don't match the current directory structure.

### Passing

`AGENTS.md` points at a recognized verification command (e.g. `charter doctor`, `moon run :check`), mentions `hk.pkl`, and matches the current toolchain and repo boundaries.

## How to fix

Update the context file to reflect the repo's current state. Check that the tech stack, test command, and edit boundaries match what's actually in the repo. Run `charter doctor` after updating to confirm the finding clears.

## Edge cases

an old date alone is not enough to fail the rule if the content is still factually accurate.
