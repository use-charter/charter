**Rule ID:** AE-CC-002 · **Severity:** High · **Category:** Agent Config · **Auto-fixable:** No

## Why this rule

Without explicit edit boundaries, an agent has implicit full-repo write access — including CI workflows, migration files, environment config, and secret stores where an unreviewed change has operational or security impact.

## What triggers it

Charter reads all tracked agent context files — `AGENTS.md`, `CLAUDE.md`, `.windsurfrules`, `.github/copilot-instructions.md`, `opencode.md`, `codex.md`, `DESIGN.md`, `SKILL.md`, every file under `.cursor/rules/`, and `PERMISSIONS.md` when present — and checks whether any of them explicitly declare off-limits paths.

The finding fires when none of those sources contains a concrete boundary listing sensitive directories. Charter looks for recognized sensitive-path tokens presented as restricted or off-limits: `.env`, `secrets`, `.github/workflows`, `terraform`, `infra`, `db/migrations`, and `credentials`. A context file that references `PERMISSIONS.md` as the authoritative boundary document also satisfies the check. A context that describes the project and stack thoroughly but never draws an edit boundary is flagged High.

This rule is intentionally stricter than AE-CTX-001, which only requires a generic mention of edit scope. AE-CC-002 requires concrete path declarations.

## Examples

### Failing

An `AGENTS.md` that documents the project architecture and commands but contains no section declaring paths the agent must not edit — flagged High.

### Passing

An `AGENTS.md` with a dedicated "Off-limits" or "Edit Scope" section listing `.env*`, `secrets/`, `.github/workflows/`, and signing keys passes. A context file that delegates to `PERMISSIONS.md` for the full boundary list also passes.

## How to fix

Add an explicit off-limits section to your agent context file — or to a `PERMISSIONS.md` that the context references — listing at minimum `.github/workflows/`, `terraform/` or `infra/`, `db/migrations/`, `.env*`, and `secrets/`. Commit the change so every agent session inherits the boundary.

## Score impact

each finding is `High` (−10); no hard cap.

## Edge cases

`.cursor/rules/` content (all tracked files concatenated) is scanned alongside the single-file context candidates; a single-purpose repo with no sensitive paths may legitimately have broad scope (documented false-positive risk); the check is presence-based on concrete sensitive-path tokens, not a semantic policy evaluation; an unreadable tracked context file fails fast (wrapped error), not silently skipped; when no agent context source exists at all, `AE-CTX-001` already fires (Blocker) and `AE-CC-002` does not duplicate the absence finding.
