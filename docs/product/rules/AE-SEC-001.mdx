**Rule ID:** AE-SEC-001 · **Severity:** Blocker · **Category:** Secrets · **Auto-fixable:** No

## Why this rule

Agent context files are read on every task. A raw credential in AGENTS.md is visible to every model, every session, and every log that captures context windows. Rotation after exposure is the only safe recovery.

## What triggers it

Charter scans agent-visible context files that are tracked in git. It applies high-confidence pattern matching for credential formats: OpenAI tokens (`sk-` prefix, 20+ chars), GitHub tokens (`ghp_` prefix, 30+ chars), AWS access keys (`AKIA` prefix, 16 chars), Slack tokens (`xoxb-` prefix, 20+ chars), and PEM private key headers.

Environment variable references (`${VAR}`, `$VAR`) and the placeholder string `your-api-key-here` are explicitly neutralized and do not trigger the rule. Only git-tracked files are scanned — uncommitted local files are out of scope by design.

## Examples

### Failing

`AGENTS.md` containing `"OPENAI_API_KEY": "sk-proj-abc123…T3BlbkFJXxyzABC"` (a real `sk-` token of length ≥ 20) — detected and redacted as `sk-p…`.

### Passing

`AGENTS.md` containing `"key": "${OPENAI_API_KEY}"` or `"placeholder": "your-api-key-here"` — both pass because env-refs and placeholder are neutralized.

## How to fix

Remove the literal credential from the file. Rotate the secret externally — assume it was exposed the moment it was committed. Replace it with an environment variable reference: `${OPENAI_API_KEY}` instead of the raw value. Commit the fix.

## Score impact

when any AE-SEC-001 or AE-SEC-002 finding is present, final Charter score is hard-capped at **49**.

## Edge cases

file must be in repo inventory AND git-tracked to be scanned; uncommitted local scratch files are safe by design; `.gitignore`d files are never scanned.
