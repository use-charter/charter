**Rule ID:** AE-CTX-004 · **Severity:** Medium · **Category:** Context · **Auto-fixable:** Yes — `charter fix --rule AE-CTX-004`

## Why this rule

Agent session files checked into git expose local machine state and session history to the repo's commit record and inflate the tracked file set every agent has to scan.

## What triggers it

Charter checks whether `.gitignore` excludes agent session artifacts: `.charter/`, `*.charter-session`, `.claude/local/`, `.cursor/cache/`. The finding also fires if any of these patterns are already tracked in git.

## Examples

### Failing

Local agent output, cache directories, or secret-bearing env files remain trackable by default — none of the expected ignore patterns appear in `.gitignore`.

### Passing

`.charter/`, `*.charter-session`, `.claude/local/`, `.cursor/cache/`, `.hk/`, and `.env*` are handled correctly without ignoring committed team contracts.

## How to fix

Run `charter fix --rule AE-CTX-004` to add the missing `.gitignore` entries. If agent artifacts are already tracked, remove them from git with `git rm --cached` before committing.

## Edge cases

`.cursor/rules`, `.claude/settings.json`, and similar team-owned config should stay committed even while local cache/state stays ignored.
