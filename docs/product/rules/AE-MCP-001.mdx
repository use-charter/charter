**Rule ID:** AE-MCP-001 · **Severity:** High · **Category:** MCP Safety · **Auto-fixable:** Yes — `charter fix --rule AE-MCP-001` (safe version bump to catalog stable; deprecated packages require manual migration)

## Why this rule

An MCP server at `@latest` can change behavior between runs without any change to the repo. A package can be deprecated, archived, or found to contain a known vulnerability after a repo first passed Charter. Pinning to an exact version ties the tool definition to a reviewable artifact.

## What triggers it

Charter scans MCP configuration files (`.mcp.json`, `.cursor/mcp.json`, `.vscode/mcp.json`) for server entries launched by package runners (`npx`, `bunx`, `uvx`, `pnpm dlx`). For each entry, Charter extracts the package spec and checks whether it pins an exact version.

Unpinned forms that trigger this rule: `@latest`, a missing version, semver ranges (`^1.0.0`, `~1.0.0`, `>=1.0.0`), floating git refs, or dynamic values like `pkg@${VERSION}`.

Charter's catalog adds additional signal: a deprecated or archived package fires High regardless of whether it's pinned. A pinned version covered by a known CVE or GHSA advisory also fires High. A pinned version that's simply behind the catalog's stable release fires informational only — it surfaces but doesn't deduct points.

## Examples

### Failing

`.mcp.json` with `"args": ["-y", "gumroad-mcp@latest"]` — floating tag, fires High.

`.mcp.json` with `"args": ["-y", "@modelcontextprotocol/server-github@1.0.0"]` — archived package, fires High with migration guidance.

### Passing

`.mcp.json` with `"args": ["-y", "@modelcontextprotocol/server-filesystem@2026.1.14"]` — exact CalVer version, catalog-known, no advisory.

## How to fix

Run `charter fix --rule AE-MCP-001` for a safe diff-first version bump. For unpinned or behind-stable packages, the fixer updates the package spec to the catalog's stable version. For deprecated packages, migration is manual — the finding includes the recommended successor.

## Score impact

deprecated/unpinned/advisory findings are `High` (−10); the behind-stable nudge is informational (0). No hard cap (caps are reserved for raw-secret findings).

## Edge cases

package specs are resolved only for direct runners (`npx`, `bunx`, `uvx`) and the `dlx` subcommand of `pnpm`/`yarn`; local path args (`./`, `/`, `../`) and the `exec`/`run` forms launch local binaries and are not treated as registry packages. Scoped packages (`@scope/name@1.2.3`) are pinned only when the trailing version is exact; `pkg@${VERSION}` is treated as unpinned; other runners are out of scope for v1.
