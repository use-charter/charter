**Rule ID:** AE-CTX-001 · **Severity:** Blocker · **Category:** Context · **Auto-fixable:** Yes — `charter fix --rule AE-CTX-001`

## Why this rule

Agents operate from what they can read in the repo. A missing or empty context file means the agent starts every session with no orientation, guesses at the stack, and may edit files it should leave alone.

## What triggers it

Charter looks for a context file (`AGENTS.md`, `CLAUDE.md`, `.cursor/rules`, or equivalent) in the repo root. The finding fires if no context file exists, if the file is empty or contains only placeholder text, or if the file exceeds the 600-token budget that agent context windows reliably fit.

## Examples

### Failing

The repo has no `AGENTS.md` or equivalent, an empty `AGENTS.md`, a file with only boilerplate like "Add your project details here", or a file that runs so long it would be truncated in the agent's context window.

### Passing

`AGENTS.md` at the repo root with a project summary, the active tech stack, a verification command (`moon run :check`), and explicit edit boundaries. Under 600 tokens.

## How to fix

Run `charter fix --rule AE-CTX-001` to scaffold a minimal `AGENTS.md`. Edit it to reflect your actual stack, commands, and off-limits paths. Keep it tight — the value of a context file degrades when it grows past the token budget.

## Edge cases

a repo using `CLAUDE.md`, `DESIGN.md`, or `SKILL.md` instead of `AGENTS.md` can pass if the file satisfies the same content requirements.
