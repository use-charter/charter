## Why this rule

A coding agent should not be pointed at arbitrary remote infrastructure without a team review. Unknown public MCP origins are a supply-chain and data-exfiltration risk â€” tool calls travel to infrastructure outside the team's visibility.

## What triggers it

Charter scans MCP configuration files (`.mcp.json`, `.cursor/mcp.json`, `.vscode/mcp.json`) for server entries that use a remote `url` or a transport type of `http` or `sse`. For each remote server, Charter extracts the URL host and compares it against the repo's `charter.yaml` `mcp.trustedRemotes` allowlist. Any public host not present in that allowlist fires High.

If no `mcp.trustedRemotes` allowlist exists in `charter.yaml`, every non-local remote server is flagged as unverifiable â€” the finding's summary makes the remediation clear: add an allowlist.

Local and internal origins are exempt from this rule. Loopback addresses, RFC 1918 private ranges, link-local addresses, and hosts under the `.localhost`, `.local`, or `.internal` TLDs are all treated as safe. A bare environment-variable URL like `${API_URL}` has no parseable host and is skipped.

## Examples

### Failing

`.mcp.json` server `"url": "https://unknown.example.net/mcp"` with no matching allowlist entry â€” flagged High; evidence names the server and host.

### Passing

`.mcp.json` server `"url": "https://mcp.asana.com/mcp"` with `charter.yaml` listing `mcp.trustedRemotes: [mcp.asana.com]` â€” host allowlisted, passes.

## How to fix

If the remote origin is intentional, add its hostname to `charter.yaml` under `mcp.trustedRemotes` and commit the change â€” this signals that your team has reviewed the server. If the origin is unknown or unintended, replace it with a trusted alternative. There is no auto-fixer for this rule because the trust decision requires a human review.

## Edge cases

local/internal origins never fire (loopback, RFC1918 private, link-local, unspecified, `.localhost`/`.local`/`.internal`); SSE (`type: sse`, deprecated in favor of HTTP) is treated as remote; a scheme-less or dynamic `${VAR}` URL has no parseable host and is skipped; allowlist matching is host-only (no scheme or path).
