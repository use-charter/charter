**Rule ID:** AE-SEC-002 · **Severity:** Blocker · **Category:** Secrets · **Auto-fixable:** No

## Why this rule

MCP configurations are read by the agent runtime before tools execute. A raw credential in mcp.json is exposed to any model or log that processes the tool-call context — and unlike source code, MCP configs are intentionally agent-visible by design.

## What triggers it

Charter scans tracked MCP configuration files — `.mcp.json`, `mcp.json`, `.cursor/mcp.json`, and `.vscode/mcp.json` — for the same high-confidence token patterns used by AE-SEC-001. It looks for literal values matching known secret prefixes: OpenAI `sk-` tokens (20+ characters), GitHub `ghp_` tokens (30+ characters), AWS access key IDs beginning with `AKIA`, Slack bot tokens starting with `xoxb-`, and PEM private key headers.

Environment variable references — both `${VAR}` and `$VAR` syntax — are neutralized before matching and always pass. Placeholder strings such as `your-api-key-here` also pass. Only literal credential values trigger the finding. Detected secrets are redacted in Charter output (first 4 characters followed by `…`).

## Examples

### Failing

`.mcp.json` containing `"api_key": "sk-proj-abc123…T3BlbkFJXxyzABC"` — a real OpenAI token of sufficient length is detected and reported as `sk-p…`.

### Passing

`.mcp.json` with `"env": {"OPENAI_API_KEY": "${OPENAI_API_KEY}"}` — the env-var reference is safe and passes. A field set to `"your-api-key-here"` also passes.

## How to fix

Remove the literal secret from the MCP config file, rotate it externally, and replace it with an environment variable reference — for example `"key": "${ENV_VAR_NAME}"`. Move runtime credentials to a secrets manager and inject them at startup rather than storing them in tracked config files.

## Score impact

when any AE-SEC-001 or AE-SEC-002 finding is present, final Charter score is hard-capped at **49**.

## Edge cases

environment variable references (both `${VAR}` and `$VAR` syntax) are intentionally safe and never flagged; only literal string values match detection.
