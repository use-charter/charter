**Rule ID:** AE-CTX-006 · **Severity:** Low (informational — does **not** deduct points or affect score) · **Category:** Context · **Auto-fixable:** No

## Why this rule

Agents that receive a high density of emphatic directives (IMPORTANT, NEVER, MUST) show measurably reduced instruction adherence. The signal degrades into noise when every sentence demands priority.

## What triggers it

Charter counts emphatic directive words — `IMPORTANT`, `NEVER`, `MUST`, `ALWAYS`, `CRITICAL`, `WARNING`, and similar — in the context file and computes a density relative to word count. The finding fires informally (no score deduction) when that density exceeds roughly 15 per 1,000 words.

## Examples

### Failing

A context file where nearly every instruction starts with `IMPORTANT:` or `NEVER`, or where multiple `CRITICAL` blocks are stacked in sequence. The finding reports the exact count and density.

### Passing

A context file that uses emphatic words selectively — reserving them for genuinely exceptional constraints rather than routine instructions. Plain declarative sentences like "Tests run with `go test`. No silent mutation. Fail fast." keep density well below the threshold.

## How to fix

Review the context file for stacked imperatives. Reserve `IMPORTANT`, `NEVER`, and `MUST` for constraints that would cause real harm if missed. Rewrite routine guidance as plain declarative sentences. The goal is signal density, not volume.

## Score impact

informational — listed in output, re-surfaces, but contributes **0** to the score (mirrors AE-SUPPRESS-003).

## Edge cases

fires only when a context file exists (absence is AE-CTX-001's concern); empty file → no finding.
